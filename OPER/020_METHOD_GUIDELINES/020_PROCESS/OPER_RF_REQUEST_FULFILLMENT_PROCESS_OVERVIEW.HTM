[RF] REQUEST FULFILLMENT

From an ITIL perspective, "Request Fulfillment is the process for managing all service requests from the users throughout their lifecycle."1

Back to Top

OBJECTIVES

From an ITIL perspective, the objectives of the Request Fulfillment process include1:

  • “Maintaining user and customer satisfaction through efficient and professional handling of service requests”.
  • “Providing a channel for users to request and receive standard services for which a predefined authorization and qualification process exists”.
  • “Providing information to users and customers about the availability of services and the procedure for obtaining them”.
  • “Sourcing and delivering the components of requested standard services”.
  • “Assisting with general information, complaints and comments”.

Back to Top

APPROACH

Request fulfillment for Oracle services is a shared responsibility between customers and Oracle. Customers can view the standard Oracle service catalog online via My Oracle Support (MOS). To fulfill formal requests for new services, contracts typically apply and Oracle sales and delivery teams fulfill such requests, including any required provisioning for hardware, software and additional required infrastructure. Customers should be aware that provisioning process details and times vary by service, so questions about the provisioning process should be directed to the Oracle SDM and/or TM.
