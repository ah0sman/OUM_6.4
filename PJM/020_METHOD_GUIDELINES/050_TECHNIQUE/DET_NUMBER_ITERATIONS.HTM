DETERMINING THE NUMBER OF ITERATIONS OVERVIEW

The purpose of this technique is to provide a means of estimating the number of iterations for each phase at any point during the project. Remember, project planning is not an exact science and initially is high-level and abstract in nature, and then becomes more detailed and precise as the project progresses. This calculation can be performed at any point during the project and should be as more accurate information becomes available.

TECHNIQUE RESULTS

Iterations per OUM Phase - The Iterations per OUM Phase is used to plan work for an OUM phase on a project.

TECHNIQUE STEPS

No. Step Component Description
1 Identify necessary factors about the project. Project Characteristics The Project Characteristics are needed in order to estimate the number of iterations. They include:
  • agreed upon time period for an iteration (4 to 6 weeks)
  • number of resources (Full-Time Equivalent or FTE) available for the project
  • number of total man-days (effort) estimated for the project
  • number of work days per week
2 Calculate the Man-Days of Effort required for an iteration within a phase. Man-Days of Effort The Man-Days of Effort provides the number of man-days per iteration or the daily burn rate for a single iteration.
3 Calculate the number of iterations that can occur within a phase. Iterations per Phase The Iterations per Phase is the number of iterations that can occur for the amount of Man-Days pf Effort estimated for the phase under consideration.

APPROACH

Identify the Necessary Factors

In order to estimate the number of iterations, the following factors must be identified:

Factor
Description
Agreed upon Time Period for an Iteration (4 to 6 weeks)
 OUM recommends an iteration length of 4 to 6 weeks. This timeframe keeps both the team and stakeholders focused on the iteration's objectives. The actual length of each iteration will vary. This agreed to time period is simply used as a factor in the calculation.
Number of Resources (Full-Time Equivalent or FTE) Available for the Project
 The number of resources available to the project is the Full-Time Equivalent (FTE) that shows how many people are available for the project.
Number of Total Man-Days (Effort) Estimated for the Project
 This factor can be derived using any available estimating approach and/or tool.
Number of Work Days per Week
 The number of work days per week is usually 5, but may need to be adjusted for local customs and/or laws.

Keep in mind, this calculation is only intended to provide a rough estimate of the number of iterations per phase. Therefore, some of the factors above may need to be approximated or assumed in some cases, particularly early on in the project.

Calculate the Man-Days of Effort

To determine the number of days required per iteration, use the following formula:

[(Number of Resources * Days per Week) * Duration of Iteration]

For example, [(10 Resources * 5 Days/Week) * 6 Weeks] = 300 Man-Days of Effort Required per Iteration.

Calculate the Number of Iterations

In this last step, calculate the number of iterations that can occur for the amount of Man-Days of Effort estimated for the phase being considered. Use the following formula:

(Number of Man-Days of Effort Estimated / Number of Man-Days Required per Iteration)

This provides a rough estimate of the number of iterations that are possible within a given phase. This result can be used as a starting point for planning the current phase and remaining project.

QUALITY CRITERIA

Use the following criteria to check the quality of this technique:

  • Have all the necessary factors been identified and verified?
  • As the project progresses, have you fine-tuned the factors?
  • Have you used an reasonable/acceptable approach/tool to calculate the Number of Total Man-Days (Effort) Estimated for the Project?
  • Have you checked your math?
