CM.020 - OBTAIN CONFIGURATION MANAGEMENT TOOLS

The purpose of this task is for the project manager with the assistance of the technical lead to procure/obtain, install and configure the tools required to support the work of Configuration Management. The requirements for these tools for the project are defined in the Configuration Management Plan and Processes component of the Project Management Plan (PMP).  Options available to the project manager to obtain the tools include but are not limited to the following:

  • Assist customer in procuring tools.
  • Obtain licenses for required tools.
  • Obtain open source tools
  • Assist customer in obtaining additional licenses for customer-owned tools.

DETAILED DESCRIPTION

If new tools are obtained or procured, the technical lead or customer team is responsible for installing and configuring the tools for the project team's use. Documentation and training materials for the tools should be uploaded to the designated project location (both physical and computer-based) and communicated to the project team.

In the event the customer does not have configuration management tools and is unwilling or unable to procure them, a risk should be created using the project procedures in the Risk Management Plan component of the PMP (BT.070). Development of software without a tool that provides basic version control and software configuration management capabilities is a high risk for the project.
