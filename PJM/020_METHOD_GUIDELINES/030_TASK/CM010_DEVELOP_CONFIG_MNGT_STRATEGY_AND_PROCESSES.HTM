CM.010 - DEVELOP CONFIGURATION MANAGEMENT STRATEGY AND PROCESSES

The purpose of this task is to document and develop the Configuration Management Plan and Processes. The Configuration Management Plan and Processes identifies the items under Configuration Management, identify the associated plans that govern those items and document any Configuration Management tools to be used as well as any designated roles and responsibilities.

ACTIVITY

PS.ACT.EG Establish Governance

Configuration Management Plan and Processes

The Configuration Management Plan and Processes is a component of the Project Management Plan (PMP) work product and identifies the items under Configuration Management, identifies the associated plans that govern those items and documents any Configuration Management tools to be used as well as any special Configuration Management roles and responsibilities.

APPROACH

Document Items Under Configuration Management

Document the items for the project under Configuration Management.

A single approach to configuration management that satisfies the requirements of all configuration items may not be practical. The Configuration Management process includes separate tasks to create plans for Project Documentation Management, Software Configuration Management, Environment and Patch Management and Software Release Management in addition to the Configuration Management Plan. In the Configuration Management Plan, identify what items will be under Configuration Management and which configuration item are associated with which governing plans, and avoid detailing all the project procedures for versioning, control and management. That information will be detailed in the appropriate individual governing plans.

Document Configuration Management Tools

List any Configuration Management tools you will be using for your project (for example, SVN, Beehive Online, etc.) and document where/how they are being used. Provide any other pertinent details, such as, Folder Structure, etc. that you wish to include in the PMP. You can also refer them to the associated Configuration Management plans for more details.

Roles and Responsibilities

If appropriate, document any pertinent information regarding special CM roles and responsibilities, for example, if the project has a designated Configuration Management Group or CM Manager, etc.

It may also be appropriate to simply refer to the Project Roles and Responsibilities presentation (STM.020) in the Project Library.
