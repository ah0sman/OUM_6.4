SM.020 - DEVELOP SCOPE CHANGE MANAGEMENT PROCESSES

The purpose of this task is to develop and document the Scope Change Management Processes to implement efficient and effective change control. The Scope Change Management Processes contains the following project procedures for monitoring and controlling project scope:

  • Scope Verification
  • Requirements Management
  • Scope Change Control

ACTIVITY

PS.ACT.EG Establish Governance

Back to Top

WORK PRODUCT

Scope Change Management Processes

The Scope Change Management Processes is a component of the Project Management Plan (PMP) work product and includes the project procedures for verifying scope, managing requirements and controlling changes to scope.

APPROACH

Define the Scope Verification Procedure

Define and document the project procedure that will be used to verify deliverables and work products, for example, Peer Reviews, Workshop Reviews, etc.

Document the Requirements Management Procedure

Document the project procedure for managing requirements. OUM recommends that requirements be managed using the MoSCoW List (RD.045).

Document the Scope Change Control Procedure

Proposed scope changes can require new or revised cost estimates, schedule activity sequences, schedule dates, resource requirements and analysis of risk response alternatives. Changes in scope, funding or deliverables that fall outside of the approved baseline scope and funding should go through formal change control.

No work should be initiated unless key stakeholders have agreed to the scope change and your appropriate corporate change control policies have been followed.

Determine Scope Change Control Steps

Identify all the steps for controlling scope during the project. Consider the following example:

  1. Identify a potential scope change.
  2. Add change to Change Request Log.
  3. Review scope change.
  4. Scope change rejected or approved.
  5. If rejected, scope change is closed and the Change Request Log is updated.
  6. If approved, Change Request Log is updated and the scope change moves forward with the appropriate corporate change management procedures.
  7. Once formally approved the scope change is implemented.
Determine the Role(s) for Each Scope Change Control Step

Once you have identified all the steps, determine what role(s) is responsible for each step.

Create Scope Change Control Flow Diagram

Optionally, create a flow diagram showing the steps and the roles.

Create or Obtain Forms

Create or obtain the following forms:

  • Change Request Form
  • Change Request Log
