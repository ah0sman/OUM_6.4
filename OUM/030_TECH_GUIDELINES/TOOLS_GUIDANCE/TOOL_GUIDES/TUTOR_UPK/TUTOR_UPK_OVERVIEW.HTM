ORACLE USER PRODUCTIVITY KIT PROFESSIONAL (UPK PRO) AND ORACLE TUTOR OVERVIEW

TOOL OVERVIEW

Oracle User Productivity Kit Professional (UPK Pro) and Oracle Tutor are complementary Oracle tools that are used to develop, deploy and maintain content that can be used throughout each phase of the enterprise application implementation lifecycle from blueprinting/design/testing to go-live and maintenance and support. UPK Pro and Tutor can be used with all of Oracle's business application brands as well as non-Oracle business applications. UPK Pro and Tutor allow project teams to streamline the creation of business process documentation, test scripts, interactive simulations, job aids, presentations, courseware, and in-application performance support, reflecting the specific business practices and applications environment. For creating the above content types, use Tutor for business process documentation and UPK for everything else.

UPK Pro's collaborative content development platform is used by the project Testing, Functional, Training and Sustaining teams to very quickly and efficiently create and maintain synchronized System Documentation, Test Scripts, Transaction Process Simulations, and in-application end-user support. Prebuilt UPK baseline content is available for sale to customers, and not only enables a rapid start on simulation development, but can be used in a functional prototype without requiring a test instance of the application to be set up. The UPK software can create over a dozen assets from one recording effort. As part of online training, users can use the simulation playback to be trained and tested on their understanding of how to complete activities that are part of their daily job.

Tutor provides content and software tools to easily create, modify, and maintain business process documentation - policies, procedures, and supporting documents. Author is a modification to MS Word, and is therefore easy to learn and use. The documentation created with Tutor depicts both the business process flow, as well as the supporting business procedure level. Tutor Author contains automated flowchart generation that makes the creation of flowcharts simple and easy. Tutor Publisher creates html based desk manuals, produces QA reports, and generates the complete html set in batch. Finally, Tutor Model Documents contain over 250 baseline procedures that enable a rapid start to developing end-user documentation.

Both UPK and Tutor content can be deployed on the organization's intranet, and in an Oracle Applications Unlimited instance. When deployed with an application, the end user can access content based on the transaction they are executing via the Help button. This provides superior in-production support to the end-user community.

Knowledge Center provides an easy-to-search knowledge portal that provides stakeholders, project managers and employees with access to a central repository of all project assets. All published UPK content, project documentation and Tutor procedures can be deployed to present a guided learning path focused on organizational roles.

By documenting the application data entry steps with UPK, employees can be provided with the exact information they need at the exact point they need it. By documenting processes and procedures with Tutor, implementing organizations are also able to provide transparency of their business practices to any other concerned audience (executive staff, internal auditors, external auditors, customers) and for formal regulatory requirements such as ISO9000, Sarbanes-Oxley, HIPPA, and SAS70.

END-USER DOCUMENTATION OVERVIEW

A successful end user documentation project requires a comprehensive plan, and Oracle has developed the End-User Documentation method for Tutor, UPK and Knowledge Center, combining long-standing UPK and Tutor Methods with the structure of Oracle Unified Method (OUM), and integrating the development of UPK Pro deliverables. The resulting approach can be integrated into an existing applications implementation project that is utilizing OUM. In addition, it can be used as a guide for a stand-alone documentation project.

End-user documentation is a critical factor in the success of applications implementations projects. A crucial project event is the development of an effective “future state model” by the project team. This future state model is a carefully defined prototype, which enables the business software to be configured to work most effectively for the implementing organization. It is crucial to communicate this to-be model to the employees within the implementing organization who will be using the applications to perform their work once the system goes live. Oracle UPK Pro and Oracle Tutor, along with this method, provide an effective means to address this challenge and prepare the end-user community to be productive on day one of go-live.

It is strongly recommended that users of this document read and become familiar with The Tutor Solution, the first chapter in the Tutor Implementation Guide.

Project End-User Documentation Work Products

Oracle UPK Pro and Oracle Tutor provide the following categories of documentation work products that can be used throughout the lifecycle of an application implementation. The primary work products are:

  • Blueprinting/Design/Testing - Capture best practices early and use presentation outputs for visual comprehension and review with subject matter experts (SMEs). Use system process documents do document process flows. Use test scripts to validate in conference room pilot. Gain key LOB/locations/Users buy in. Use "See it" mode with voice explanations. Walk through process flows using presentation outputs or system process documents.
  • Business Process Documentation - Tutor Procedures are role-based work instructions that guide the end user in the proper use of business systems in their organization. Procedures are deployed as html and can be linked to from within the application environment, creating a living set of business process documentation easily accessible to the end-user community. Tutor also provides a hierarchy of process flows and sub processes that drill down into the procedures. The UPK documents are MS Word-based outputs of transaction recordings, and include system process and test documents.
  • Test Plans - With UPK, capture test steps automatically and them import those steps directly into Oracle Application Testing Suite, IBM Rational or HP Quality Center testing suites eliminating many of the errors that occur when writing manual tests. These test scripts, along with system process and test documents, decrease the testing time and costs during conference room pilots and shorten the project timeline.
  • Transaction Recordings - UPK develops annotated transaction recordings that depict how Oracle Applications can be executed on a field-by-field basis. The implementing organization develops the transaction recordings to show exactly how a user would perform a transaction in their environment. MS Word-based system process documents, test documents, job aids, instructor manuals and training guides are co-products of the transaction recordings. Like procedures, transaction recordings can be linked to from the application environment, and procedures and transaction recordings can link to each other.
  • Oracle Applications Help Documents - Oracle Applications contain Help documentation deployed as context sensitive html. When an end user is in a specific transaction and presses the help button, a help document opens that applies to that transaction. Oracle UPK transaction recording content can be integrated directly into the application help system, providing end users context-sensitive help and just-in-time support and information.
  • Courseware - The project content work products created with UPK can be leveraged to create self-paced or classroom training materials for instructor-led training (ILT).
  • Guided Training Portals - Knowledge Center, a component of UPK Pro, creates an easy-to-search knowledge portal that provides stakeholders, project managers and employees with access to a central repository of all project assets.

Documentation Project Resource Requirements

Documentation projects require specialized expertise and dedicated resources. The tools are easy to learn, although not all resources in a project need to learn the complete tool set. The following resources are needed for such a project. Project teams may be composed of resources from consulting organizations, from the customer organization, and from third-party contractors hired for the effort. The customer organization is advised to take on as many of these execution roles as resources allow, ensuring a higher degree of success in the long term.

  • Executive Sponsor - Customer executive providing vision and authority
  • Project Manager - Project manager from the consulting organization
  • Customer Project Manager - Project manager from the customer organization
  • Applications Specialist - Oracle Applications functional expert
  • Tutor & UPK Consultant – Tutor and/or UPK experts from the consulting organization or Oracle University
  • Document Owner - Subject matter expert from the customer user community who has the authority to determine how a process operates, and the responsibility to make sure it continues to meet business needs today and into the future
  • Document Controller - Tutor Publisher expert who manages QA and publishing
  • Document Specialist - Tutor Author expert
  • UPK Developer - Oracle UPK expert or SME
  • Developer - Create test scripts with UPK
  • Business Analyst - Create test scripts with UPK
  • Tester - Conduct tests using with UPK scripts and documents
  • Course Designer - Knowledge Center development expert
  • Instructor - Oracle University Tutor or UPK expert
  • Department Manager - Customer manager whose organization is being impacted by change
  • Auditor - External or internal auditor
