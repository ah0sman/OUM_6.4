DEFINITIONS AND TERMS

A

Account
An organization or company that is a current or potential customer.

Account Briefing
See ACTIVE LANGUAGE.

ACD
See AUTOMATED CALL DISTRIBUTION.

Action
An event caused by the execution of an SQL statement, an external program, or both.

Active Language
The active language in the client's user interface when starting a Siebel application, as used and defined by the Language parameter in the .cfg file or by the /L command-line parameter.

Actuate
A report writer from the Actuate Software Corporation. The report writer consists of a viewer tool and a report design tool (the Actuate Developer's Workbench), which are integrated into Siebel applications.

Advisory Services
A term that refers to acquisitions and mergers in Siebel Finance.

Agent
A call center representative who places outbound calls, answers inbound calls, or both — such as for sales or support purposes.

ANI
See AUTOMATIC NUMBER IDENTIFICATION.

API
See APPLICATION PROGRAMMING INTERFACE.

Applet
1.  In Siebel applications, a visual application unit that appears on screen as part of a view. 2.  In Internet applications, a self-contained program unit. See also APPLET OBJECT TYPE.

Applet Designer
A visual design editor in Siebel Tools that allows developers to create or modify Siebel applets interactively. It is used to modify and extend list, form, tree, and chart applets, and the controls and list columns on the applets.

Applet Object Type
A Siebel object type that implements a visual application unit (an applet) that appears on screen as part of a view. An applet serves as the vehicle for data entry and navigation through the data, and may implement a data entry form, scrolling list table, chart, dialog box, navigation tree, or Web browser. It provides viewing, entry, modification, or navigation capabilities for data in one business component. See also BUSINESS COMPONENT OBJECT TYPE and VIEW OBJECT TYPE.

Application
A program or group of programs designed for end users. In Siebel products, an application is a subset of the Siebel Applications that can be invoked from an icon on the desktop or Start menu; for example, Siebel Sales or Siebel Service. See also APPLICATION OBJECT TYPE.

Application Administrator
People responsible for planning, setting up, and maintaining their company’s Siebel applications. See also DATABASE ADMINISTRATOR and SYSTEM ADMINISTRATOR.

ApplicatioN Object Type
A Siebel object type. Each application is a defined subset of the constructs available in Siebel Applications. An application presents a grouping of screen object definitions.

Application Programming Interface
An API is a set of routines used by an application program to direct the performance of procedures by the computer’s operating system.

Application Upgrader
A utility within Siebel Tools that allows developers to merge a repository (database of object definitions) from an older version of Siebel Applications, including one that a developer may have significantly customized, with a newer version. See also REPOSITORY.

Argument
1.  An independent variable, used with an operator or passed to a subprogram that uses the argument to carry out specific operations. 2.  An argument can also be an option to a command, in which case it is often called a command-line argument.

Association Applet
This applet provides the user with the ability to associate records of two business components that have a many-to-many relationship. An association applet is invoked from the Edit Add New Record menu option in a master-detail view.

Association Table
A synonym for intersection table. See INTERSECTION TABLE.

Asynchronous
Not synchronized; that is, not occurring at predetermined or regular intervals. This term is usually used to describe communications in which data can be transmitted intermittently rather than in a steady stream or when an immediate response is not necessary or possible.

Attachment Applet
See FILE ATTACHMENT APPLET.

Attribute
Sometimes used as a synonym for property. See PROPERTY.

Authentication
The process of identifying an individual, usually based on a username and password. Authentication merely ensures that the individual is who he or she claims to be, but says nothing about the access rights of the individual.

Automated Call Distribution
Telephony hardware (such as a switch) and software that work together to route calls to call queues and call center agents.

Automatic Number Identification
Technology that passes a caller’s phone number along with the call. Through database lookup, for example, a caller or the caller’s company can be identified as a Siebel customer.

B

Base Column
A column in a database table in Siebel Tools that is used in Siebel Applications. The base columns contain descriptive information for primary or foreign keys that are defined and used by Siebel Applications. A base column is not necessarily a column in a base table.

Base Table
1.  A table containing columns that are defined and used by Siebel applications. 2.  A table that provides the essential columns to a business component; specified in the Table property of the business component. See also EXTENSION TABLE and TABLE OBJECT TYPE.

Basic Calls Window
A window that displays a list of Siebel VB subroutine and function calls that were executed prior to the current line. It is found in the Siebel VB Debugger. See also Siebel VB DEBUGGER.

Bookmark
A feature that uses Microsoft Word’s internal field capabilities. A bookmark is used as a reference point within a Word document. For example, within Siebel Proposal Generator, you can map a Business Component field to a Word bookmark. The bookmark name in a briefing component must match the bookmark name referenced within Siebel exactly. It can only reference the current record within a business component. It cannot sum a series of records, and it cannot combine information from more than one field.

Boolean
1.  A data type in Siebel Tools for a field or column where the values for that field or column can be either TRUE or FALSE. Developers specify a data type of DTYPE_BOOL to define a field as Boolean. 2.  In the Siebel Object Types Reference , Boolean is one of the values that may appear in the Required column in the list of properties for an object type. A value of Boolean for a property, as opposed to Required, Optional, or System, indicates that it is a TRUE/FALSE property that will retain its default value if the developer does not specify TRUE or FALSE. Also called logicaloperator. See also OPERATORS and RELATIONAL OPERATORS.

Boolean Operators
Boolean operators enable you to test the truth or falsity of conditions. See also operators and relational operators. Briefing Dynamic reports that Siebel eBriefings creates by integrating information from multiple internal and external sources to give users targeted, up-to date business intelligence. A standard installation of Siebel eBriefings includes My Briefings, Account Briefings, Competitor Briefings, and Company Briefings.

Bulletins
Company broadcast messages published in My Briefings to alert all users, or users associated with specific positions and divisions, to internal company announcements.

BusComp
An abbreviation for business component, and the name of the object variable used to reference an instance of a business component in Siebel VB and the Siebel Object Interfaces.

Business Component Object Type
A Siebel object type that defines a logical representation of columns in one or more database tables. A business component collects columns from the business component’s base table, its extension tables, and its joined tables into a single structure. Business components provide a layer of abstraction over tables. Applets in Siebel Applications reference business components; they do not directly reference the underlying tables.

Business Object
See BUSINESS OBJECT DEFINITIONS and BUSINESS OBJECT OBJECT TYPE.

Business Object Component Object Type
A Siebel object type that is used to include a business component in a business object. It is a child of the object type called Business Object.

Business Object Definitions
Object definitions that reside in the Business Objects layer of Siebel Applications. They are built on data object definitions and selectively combine and associate data object definitions into logical data constructs that are useful for application design. Some of these logical constructs are business components, fields, links, multi-value links, and business objects. See also BUSINESS OBJECT OBJECT TYPE and BUSINESS OBJECTS LAYER.

Business Object Interfaces
See SIEBEL OBJECT INTEFACES.

Business Object Manager
A synonym for Business Objects Layer. See BUSINESS OBJECTS LAYER.

Business Object Object Type
A Siebel object type that creates a logical business model using links to tie together a set of interrelated business components. The links provide the one-to-many relationships that govern how the business components interrelate in this business object.

Business Objects Layer
The architectural layer in the repository that holds business object definitions in Siebel Applications. The object definitions represent database data more abstractly than the object definitions in the Data Objects Layer, and define interrelationships between data object definitions. For example, some object types in this layer are Business Object, Business Component, Link, Field, Join, and Pick Map. See also BUSINESS OBJECT MANAGER, DATA OBJECTS LAYER and USER INTERFACE OBJECTS LAYER.

Button
A control with a Type property value of PushButton. It initiates an action when clicked.

C

CA
See CERTIFICATE AUTHORITY.

Call Script
In Siebel SmartScript, an ordered set of questions and the transitions between them as defined by a Siebel applications administrator and used by a call center agent. A question represents a single interaction between a call script and the call center agent. An answer is a specific value defined for a question.

Cascading Menu
A submenu. Also called a hierarchical menu or child menu.

Case Sensitive
A program's ability to distinguish between uppercase (capital) and lowercase (small) letters. Programs that distinguish between uppercase and lowercase are said to be case sensitive.

Catalog
The hierarchy of products and categories in a model. See also CATEGORY, PRODUCT and MODEL.

Category
A group of related items in Siebel Product Configurator. A category can include any combination of products and other categories. Categories are defined in the Configuration Designer screen to organize the end user’s configuration choices in Siebel Quote Assistant. See also PRODUCT CONFIGURATOR.

Certificate Authority
A trusted third-party organization or company that issues digital certificates used to create digital signatures and public-private key pairs. The role of the CA in this process is to guarantee that the individual granted the unique certificate is, in fact, who he or she claims to be.

Channel Changer
The navigational aid that is visible in Siebel InterActive when accessed from a standard Web browser over the Internet. The Channel Changer consists of a number of buttons. Each button accesses a Siebel InterActive view or tool, such as an ActiveBriefing view or the Tracking Profile tool.

Channel Partner Company
A company that sells another company's products, such as a retailer, a distributor or a value-added reseller.

Chart
A graphic or diagram that displays data or the relationships between sets of data in pictorial rather than numeric form. See also CHART APPLET and CHART OBJECT TYPE.

Chart Applet
A style of applet that displays a business graphic. It is dynamically populated with data from the applet’s associated business component.

Chart Object Type
A Siebel object type that implements a business graphic, such as a line graph, bar graph, or pie chart, in a chart applet. Properties in a Chart object definition control the acquisition, conversion, and presentation of data from the business component. Chart is a child object type of the Applet object type.

Check Box Control
A control in a form applet consisting of a small square into which an “Xmark” can be entered or cleared by clicking. It represents TRUE/FALSE data. A check box is implemented as a control with a value of CheckBox in the Type property.

Check In (v), Check-In (n)
In Siebel Tools, to move the object definitions in one or more projects from a developer’s client-side repository to the server repository. A check-in also clears the server lock on the checked-in object definitions, making these object definitions available for use by other developers.

Check Out (v), Check-Out (n)
In Siebel Tools, to move the objects in one or more projects in the server repository to a developer’s client-side repository. Check-out also places a lock on the checked-out object definitions, making these object definitions unavailable to other developers.

Child
The hierarchically subordinate member of a special relationship between two object types or object definitions: the parent-child relationship. A child object type has exactly one parent object type; a parent object type may have one or more child object types. For example, Site is a Field object definition that is a child of the Business Component object definition called Account. Similarly, a child object definition has exactly one parent object definition. For example, the Site Field object definition is a child object definition of the Account Business Component (parent) object definition.

Class
In object-oriented programming, a generalized category that describes a group of more specific items, called objects, that can exist within it. A class is a descriptive tool used in a program to define a set of attributes or a set of services (actions available to other parts of the program) that characterize any member (object) of the class.

Class Property
A property in Siebel Tools that is used in certain object types, such as Applet and Business Component. Selecting a Class property value assigns a style to the object definition, which dictates behavior and look and feel. For example, a value of CSSFrameList in the Class property in an Applet object definition makes it a list applet. There is no strict correspondence between class in the object-oriented programming sense and the Class property in an object definition in Siebel Tools.

Client
A computer that uses the services of other computers and accesses the data provided by other computers. Siebel applications support three types of clients, dedicated, mobile, and thin. See DEDICATED CLIENT, MOBILE CLIENT and THIN CLIENT.

Client/Server Architecture
A network architecture in which each computer on the network is either a client or a server.

Client Wins
A rule used to resolve a data conflict in Siebel Remote during a data merge, and Enterprise Integration Manager (EIM) and repository merges. During the merges, the data value from the client database takes precedence over the data value on the server database. To ensure consistent results, both client and server must employ the same rule. See also ENTERPRISE INTEGRATION MANAGER and SERVER WINS.

Cluster Services
A server platform that allows you to run Siebel Servers and the Gateway Server on two-node hardware clusters for high levels of availability and fault tolerance. See also CLIENT/SERVER ARCHITECTURE.

Column
A relational database structure that specifies a particular item of information to include in each row in a database table. It also specifies data type and length. See also COLUMN OBJECT TYPE.

Column Object Type
A Siebel object type that maps to an underlying database table column. Column is a child object type of the Table object type.

Combo Box
1.  In Windows terminology, a control that combines a text box and an interdependent list box. 2.  In Siebel applications, combo box functionality is usually implemented by using a text box control that is configured as a static picklist. However, there is also a combo box control used primarily in chart applets. See also STATIC PICKLIST.

Combo Chart
A type of chart applet that displays a line graph overlaid on a bar chart.

Command
1.  An instruction to a computer program that, when issued by the user or a program, causes an action to be carried out. 2.  A command in Siebel CTI is an instruction, such as to make a call, that is sent from the Siebel client to the CTI middleware, for execution on the telephone switch.

Command Object Type
A Siebel object type that specifies the method invoked when a toolbar button associated with this command is executed. It also defines which bitmap appears on the toolbar button.

Common Object Request Broker Architecture
Also called CORBA, which is an architecture that enables pieces of programs, called objects, to communicate with one another regardless of what programming language they were written in or what operating system they are running on.

Component
A small binary object or program that performs a specific function and is designed in such a way as to easily operate with other components and applications.

Compound Query
In Siebel Tools, a selective search for object definitions in the Object List Editor that combines two or more search conditions.

Computer Telephony Integration
CTI is the means by which a computer system can interact with the telephone system. Telephony activity, such as making a call or transferring a call, can be initiated within a computer program such as Siebel Applications. Information about calls, such as ANI, DNIS, or call status data, is passed from the switch to the computer system. With CTI, the computer system supplements and extends the capabilities of the telephone switch—for example, by helping to route incoming calls to the most appropriate agent, or by presenting relevant customer records to agents taking calls. Siebel CTI works with other components to provide CTI capabilities for users of Siebel Applications.

Configuration
Typically in Siebel Tools, the modification of Siebel applications to change the look, feel, and behavior to meet specific business processes.

Configuration File
1.  A Siebel file (extension .cfg) through which application configuration data is passed to the Siebel client program. 2.  An automatically generated file containing data about the model. The name of this file is entered when a model is created. Configuration files are required by Siebel Quote Assistant and the Model Validation view.

Conflict Detection
The process of identifying inconsistent database change transactions between intermittently connected databases. For example, two updates with differing values to the same field of a table would cause a conflict. Siebel Remote detects duplicate conflicts, operation conflicts, and update conflicts. See also CLIENT WINS, MOBILE CLIENT, SERVER WINS and UPDATE CONFLICT.

Conflict Resolution
The process of applying a rule to conflicting transactions to determine which transaction has precedence for updating the database. See also Client Wins, Server Wins, and update conflict.

Control
A visual object displayed in a graphical user interface. It provides user interaction with the application. See also CONTROL OBJECT TYPE.

Control Object Type
A Siebel object type that implements a data control, typically in a form applet, with which the user can interact. Examples include text boxes, check boxes, and buttons. Generally, a Siebel control exposes data from one field in the business component or invokes programming logic (in the case of a button control). See also FORM APPLET.

Control Toolbar
In Siebel Tools, a toolbar that becomes active when the developer displays the Applet Designer. The buttons in the Control toolbar correspond to controls that can be added to the applet in the Applet Designer. See also A visual design editor in Siebel Tools that allows developers to create or modify Siebel applets interactively. It is used to modify and extend list, form, tree, and chart applets, and the controls and list columns on the applets.

CORBA
See COMMON OBJECT REQUEST BROKER ARCHITECTURE.

CTI
See COMPUTER TELEPHONY INTEGRATION.

Customization
A term that refers to the modified code of underlying object definitions to change the look and feel of an application. Siebel source code is not distributed as part of the product, and thus, cannot be customized. However, some customization is possible using Siebel VB. See also configuration and Siebel VB.

D

Database
A collection of logically related data designed to meet a business need.

Database Administrator
An individual whose duties might include responsibility for the design, management, and implementation of the database. See also APPLICATION ADMINISTRATOR and SYSTEM ADMINISTRATOR.

Database Extension Designer
A utility in Siebel Tools that allows the developer to add columns and indexes to tables in Siebel Applications, or add new tables where extensions are required beyond the database tables as delivered.

Database Extract
The database extract component of Siebel Remote and Replication Manager that creates a database snapshot file for a given mobile user. The snapshot contains all of the data required to initialize the user’s local database. A database extract retrieves data according to predefined routing rules.

Database Initialization
The process used to import data into an empty local database.

Database Management System
A software interface between the database and the user.

Database Server
The computer on which a shared database resides; the combination of hardware and software that enables users to access information stored in the database.

Data Management or Data Manager Layer
A synonym for Data Objects Layer. See also DATA OBJECTS LAYER.

Data Manager
A synonym for Data Objects Layer. See also DATA OBJECTS LAYER.

Data Mart
Data marts, which focus on a particular subject or department, are subsets of data warehouses. The larger data warehouse combines databases across an entire enterprise.

Data Model
See SIEBEL DATA MODEL.

Data Object Definitions
Object definitions that reside in the Data Objects layer. They directly map the data structures from the underlying relational database into Siebel Applications, providing access to those structures by object definitions in the Business Objects layer. Data object definitions create a layer of abstraction over the DBMS, insulating the application and the developer from database administration and restructuring.

Data Objects Layer
The architectural layer in the repository that consists of data object definitions. These include object definitions that directly represent tables, columns, and indexes in the underlying DBMS. They include the Table, Column, Index, and Index Column object types.

Data Security
A scheme for protecting data from intentional or unintentional update or deletion. Often, the protection is done through granting specific privileges to each user to create, read, update, and delete specific information.

Data Table
A Siebel table with a Type property value of Data (Public) or Data (Private). Data tables are in contrast to extension and interface tables, among others. See also interface table and table.

DBA
See DATABASE ADMINISTRATOR.

DBMS
See DATABASE MANAGEMENT SYSTEM.

Debugger
See SIEBEL VB DEBUGGER.

Dedicated Client
A computer that is directly connected to a high-speed, high-bandwidth network, able to communicate in real time with its servers. Typically, a stationary client is a desktop workstation located close to its server and benefits from rapid response times to service requests. See also MOBILE CLIENT.

Default
An operation or value that the system or application assumes, unless a user makes an alternative choice. The default is built into the program when a value or option must be assumed for the program to function.

Destination Column
The column in the destination table in a join specification. The join operation is performed on this column. It is pointed to by the foreign key (source field) in the source table. See also FOREIGN KEY COLUMN.

Destination Table
The table in a join that is joined to a business component’s base table. See also BUSINESS COMPONENT OBJECT TYPE and JOIN OBJECT TYPE.

Detail
Pertains to table rows or business component records. In a one-to-many relationship, the detail records or rows correspond to the “many.” For example, in a master-detail view displaying an Account form applet at the top and a Contact list applet at the bottom, the records in the Contact list applet are the detail records. See also MASTER.

Detail View Option
In Siebel Tools, one of three ways in which the object types can be displayed in the Object Explorer. When the Detail view option is selected, object definition names are displayed in the tree structure beneath their object types. See also OBJECT EXPLORER.

Details Visualization View
One of three visualization views in Siebel Tools. It is a diagram that displays how business components map to other business components and underlying tables. See also FIELD OBJECT TYPE, HIERARCHY VISUALIZATION VIEW, and RELATIONSHIP VISUALIZATION VIEW.

Dialed Number Identification Service
Technology that passes the phone number a caller dialed along with the call. In a call center handling multiple numbers such as 800 numbers, each of which may serve a different company, product line, or campaign, the DNIS number associated with a call can help determine the purpose of the call, and can be used to route the call.

Dialog Box
A box that appears on a display screen to present information or request input. Typically, dialog boxes are temporary—they disappear once the user has entered the requested information.

Directory
A way of organizing and grouping files into a hierarchical structure. The topmost directory is the root directory; the directories within a directory are subdirectories. See also SUBDIRECTORY.

Display Value
The value that is displayed to the end user. This value is language dependent. See also VALUE.

Distributed Database
A database that consists of two or more data files located at different sites on a computer network. The distributed database allows different users to access it without interfering with one another. Periodically the DBMS must synchronize all the databases to make sure that they all have consistent data.

DNIS
See DIALED NUMBER IDENTIFICATION SERVICE.

Domain
A group of computers and devices on a network that are administered as a unit with common rules and procedures. Within the Internet, domains are defined by the IP address. All devices sharing a common part of the IP address are said to be in the same domain. See also INTERNET PROTOCOL ADDRESS.

Domain Name
A name that identifies an IP address. For example, the domain name siebel.com represents the IP address 209.37.100.19. Domain names are used in URLs to identify particular Web pages. For example, in the URL http://www.siebel.com, the domain name is siebel.com. See also Internet protocol address. Domain Name System Every Web server requires a Domain Name System (DNS) server to translate domain names into IP addresses. A DNS is necessary because the Internet is based on IP addresses, not domain names. See also INTERNET PROTOCOL ADDRESS.

Drilldown (n), Drill Down (v)
A term that describes the movement from one view to another view that has greater detail.

Duplicate Conflict
A conflict that occurs when two database rows residing on different machines have the same value for the user’s primary key.

E

Edit Toolbar
A floating toolbar in the Siebel VB Editor that provides options for manipulating the text in the Editor window, such as cut, copy, paste, and save.

EIM
See ENTERPRISE INTEGRATION MANAGER.

Ellipsis
Symbol consisting of three dots in a row (…) seen on a button or menu option. When you click an ellipsis button, a dialog box appears.

Enterprise Integration Manager
The EIM manages the exchange of data between Siebel database tables and other corporate databases. It is used to perform bulk imports, exports, merges, and deletes. See also INTERFACE TABLE.

Escalation Policy
A set of actions that are performed when a specific set of conditions is met.

Event
1. An action or occurrence t o which a program might respond, such as key presses, button clicks, or mouse movements. 2.  Siebel events refer to selected events within Siebel Applications that allow the association of extension routines that extend the base behavior of the application. When a Siebel event is “fired” or activated, the developer-specified procedures are invoked along with the standard procedures coded in Siebel Applications. 3.  In Siebel CTI, an event is an occurrence, such as a call ringing, that is sent from the telephone switch to the CTI middleware, and possibly on to a Siebel client.

Extensible Markup Language
See XML.

Extension
A single line or directory number available for a teleset. The number or types of extensions available per teleset depend on the manufacturer.

Extension Column
A column that is not used by standard Siebel Applications, but is used by a custom-configured application. There are three kinds of extension columns: standard extension columns (included for use by developers), custom extension columns in an extension table (columns added by the developer to an extension table), and custom extension columns in a base table (added by the developer to the base table of an extension table). See also BASE TABLE and EXTENSION TABLE.

Extension Table
A table in Siebel Applications that provides additional columns to a data table that cannot be directly added to the original table. The database product may only support a limited number of columns, or it may not allow adding a column to a table once it is populated with data. Siebel Applications provide extension tables for most of its base tables. See also BASE TABLE and EXTENSION COLUMN.

External Content
Information and news that is created outside of the user’s company and is brought to the user by Siebel eBriefings through arrangements with external content providers (independent information providers).

F

Field
1. A location in a record in which a particular type of data is stored. 2.  Informally, an area for data display and entry in a Siebel applet, used synonymously with control or list column. See also FIELD OBJECT TYPE.

Field, Calculated
1. A calculated field is one whose value is derived from some formula involving other fields. You do not enter data into a calculated field; the system automatically determines the correct value. 2.  A calculated field in Siebel Tools is implemented by the specification of a Calculated Value property value in a Field object definition. See also FIELD OBJECT TYPE and PROPERTY.

Field Object Type
A Siebel object type that maps a column to a business component. A field provides a meaningful name that does not require knowledge of the base table or joined table of its origin. Fields supply data to controls and list columns in the user interface. See also COLUMN OBJECT TYPE, CONTROL OBJECT TYPE and FIELD, CALCULATED.

File
A collection of data or information that has a name, called the filename. Almost all information stored in a computer must be in a file. There are many different types of files: data files, text files, program files, directory files, and so on.

File Attachment Applet
A style of applet, typically a list applet. It provides access to external documents from within Siebel applications and allows the user to open, add, edit, or remove documents from the list applet. It also provides synchronization and shared access support for attached documents. See also APPLET OBJECT TYPE.

Firewall
A system designed to prevent unauthorized access to or from a private network. Firewalls can be implemented in both hardware and software, or a combination of both.

Flat-File Database
A simple database system in which each database is contained in a single table. Relational database systems can use multiple tables to store information, and each table can have a different record format.

Flat View Option
A viewing option in the Object Explorer that displays all object types without their hierarchical relationships. Invoked by clicking the Flat tab.

Foreign Key Column
A column in a database table that is used to store the values of a primary key column of another table.

Form Applet
A style of applet in Siebel applications that displays data in a data entry form. It displays one record at a time from the underlying business component. Fields in the business component appear on the form applet as text boxes, check boxes, and other controls.

Graphical User Interface
A GUI is a type of environment that represents programs, files, and options by means of icons, menus, and dialog boxes on the screen. Some of the GUI items found in Siebel applications include the view bar, toolbar, and tab bar, and various dialog boxes, controls, and buttons.

Group Control
A rectangular box that is drawn on the applet and serves as a visual aid for organizing the controls in the applet. It does not have data-editing or display capabilities.

GUI
Pronounced goo-ee. See GRAPHICAL USER INTERFACE.

H

Helpld
A value specifying a topic name for context-sensitive Help. To make context-sensitive Help work for an applet, the applet needs to pass the HelpId value to Windows Online Help.

Hierarchy Visualization View
In Siebel Tools, one of three visualization views. It is a diagram that displays the parent-child relationships between the selected object definition, and its parent and child object definitions, and their parent and child object definitions (and so on). See also DETAILS VISUALIZATION VIEW and RELATIONSHIPS VISUALIZATION VIEW.

Home Page
The starting point of a Web site. Also, the Web page that a browser is set to use when the user first starts it.

HTML
See HYPERTEXT MARKUP LANGUAGE.

HTML Application
A Siebel Repository representation of a Siebel HTML Thin Client-based Application as an object of type Application. It contains definitions of the various Applets, Views, Web pages, and so forth that make up the Application.

HTML Template
An HTML model or style file for generating a Web Template containing both standard HTML and Siebel Tags. Information in the HTML Template file is combined with information stored in the Web Template Object definition in the repository to generate the final HTML output.

HTTP
See HYPERTEXT TRANSFER PROTOCOL.

HTTPS
See SECURE HYPERTEXT TRANSFER PROTOCOL (S-HTTP).

Hypertext
A method of presenting textual information online so that users can follow built-in links (provided by the author) to take them from topic to topic. Rather than being restricted to viewing information in a certain sequence, users are free to choose which links they want to follow in order to browse through related topics in the order they choose. Siebel supports hypertext links to supplement navigational techniques and integration with attachments.

Hypertext Markup Language
HTML is the coding language used to create hypertext documents for use on the World Wide Web. In HTML documents, blocks of text are surrounded by codes (called tags) that indicate how the text should be displayed and where links are placed. HTML is platform-independent, meaning that the codes can be read by Web-browser programs on any operating system.

Hypertext Transfer Protocol
HTTP is the communications standard used to connect World Wide Web sites and to move hypertext files across the Internet.

Identifier Code
A unique code in Siebel InterActive used by a content provider to label a company about which it provides news and other information. Each external content provider uses a different coding scheme to label the companies it tracks.

Image Control
An image control in an applet lets the developer display pictures stored as bitmap files, which are associated with business component records.

Import
