ENTERPRISE SECURITY SUPPLEMENTAL ACTIVITY GUIDELINES

The Activity Guidelines are provided to assist the project team in performing certain key tasks in an OUM-based Enterprise Security project. OUM defines an activity as "a set of tasks related either by topic, dependencies, data, common skills or work products."

Currently, guidance is included for nine Enterprise Security domains. These domains are in-line with the ISC2 definitions, but have been renamed slightly to be more in line with the Oracle identity management, and security products. The following Enterprise Security domains are included:

  • Identify and Access Control (I&AC)
  • Infrastructure Security (IS)
  • Information Security Governance and Risk Management (ISG&RM)
  • Application Security (AS)
  • Data Security and Cryptography (DS&C)
  • Security Architecture and Design (SA&D)
  • Security Operations Management (SOM)
  • Business Continuity and Disaster Recovery (BC&DR)
  • Regulatory Compliance (RC)

Use the Enterprise Security supplemental activity guidance provided on the left side of this table, in conjunction with the standard OUM Activity overview on the right side of the table, in order to understand how Enterprise Security impacts specific OUM activities.
