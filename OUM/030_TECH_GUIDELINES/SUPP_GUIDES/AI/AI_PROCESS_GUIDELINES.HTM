APPLICATION IMPLEMENTATION PROCESS GUIDELINES

This document contains OUM supplemental process guidance for Application Implementation projects.

Keep in mind that not ALL OUM tasks are included in the Application Implementation Views. Refer to the Requirements-Driven Application Implementation View and the Solution-Driven Application Implementation View for a complete list of included tasks for each view.

BUSINESS REQUIREMENTS (RD) PROCESS GUIDANCE

Approach

In a commercial off-the-shelf (COTS) application implementation, business requirements are used as input to the high-level Future Process Model (RD.011.1) and subsequently the more detailed Future Process Model (RD.011.2).

During Business Requirements, you:

  • Examine current processes and practices to understand and document the main business factors that currently benefit the business, if in scope for your project.
  • Carefully document audit and control requirements to satisfy financial and quality policies.
  • Identify the business operating requirements that the technical architecture will need to support.

The Use Case Specification (RA.024), which is prepared in the Requirements Analysis (RA) process, defines the necessary steps for executing a business function within a business process; some of the steps may be application-specific, and some may be manual. To confirm that business needs are met, you will also later map these requirements against application functions and features in Analysis.

REQUIREMENTS ANALYSIS (RA) PROCESS GUIDANCE

Approach

In a commercial off-the-shelf (COTS) application implementation, business requirements reflected in the Future Process Model (RD.011.2) are refined and structured via a Use Case Model in order to establish a more precise understanding of the requirements.  At a minimum, Use Case Models and Use Case Specifications should be prepared for those requirements that can only be satisfied by custom-built components, which extend the functionality of the COTS system and/or integrate the COTS system with other COTS or legacy systems. Preparation of Use Case Models and Use Case Details for all other business requirements is considered best practice, as it will contribute to a better understanding of requirements and can be leveraged to prepare a number of other required work products downstream, such as test scenarios and business procedures documentation.

ANALYSIS (AN) PROCESS GUIDANCE

Approach

The following tasks are generally performed only when there is a need to gain further clarification of the business requirements represented in the use cases:

  • Analyze Data (AN.050)
  • Analyze Behavior (AN.060)
  • Analyze User Interface (AN.090)
  • Prepare Analysis Specification (AN.100)
  • Review Analysis Model (AN.110)

This typically applies to those requirements that can only be satisfied by custom-built components, which extend the functionality of the COTS system and/or integrate the COTS system with other COTS or legacy systems.
