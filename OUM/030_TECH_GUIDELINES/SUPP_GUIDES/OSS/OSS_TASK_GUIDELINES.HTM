TECHNICAL ARCHITECTURE (TA) PROCESS GUIDANCE

TA.060 Define System Operations and Management Strategy

My Oracle Support access and use should be a key component of the System Operations and Management Strategy. Its proper employment, in conjunction with other operations management tools and procedures, can contribute significantly to the establishment of a personalized, proactive and collaborative environment for managing the system. From an ITIL perspective, the definition of the System Operations and Management Strategy is typically done within the ITIL Service Design Lifecycle phase. To ensure effective use of My Oracle Support the System Operations and Management Strategy should address the following recommended best practices contributing to the lifecycle management of the My Oracle Support Services portal:

  • Establish Policies for My Oracle Support Browser Usage - Establish process and procedures for managing the My Oracle Support browser requirements. Plan the installation, configuration and maintenance of the browsers. Consider alternative solutions for use of browser functionality within the oracle.com domain. Refer to the Establish Browser Policies section for guidance on establishing browser policies.
  • Define Oracle Configuration Manager Configuration - Implement Oracle Configuration Manager to centralize configuration information. Oracle Configuration Manager tracks key Oracle system details, providing essential data to manage the Oracle system environment. Oracle Configuration Manager is used to personalize the support experience by collecting configuration information and uploading it to the Oracle repository. When customer configuration data is uploaded on a regular basis, customer support representatives can analyze this data and provide better service to the customer. For example, when a customer logs a service request, the configuration data can be associated directly with that service request. The customer support representative can then view the list of systems associated with the customer and solve problems accordingly. The Oracle Configuration Manager Installation and Administration Guide provides configuration guidance.
  • View Oracle Configuration Manager Best Practices - View Best Practices to understand how Oracle Configuration Manager can be used to maintain system health. The Oracle Configuration Manager Quick Start Guide contains best practices for the lifecycle management of Oracle Configuration Manager.

TA.090 Develop Security and Control Strategy

When implementing any new system involving the installation of software, there is always a concern that line of business security may be compromised. My Oracle Support access and use should be a key component of the Security and Control Strategy. From an ITIL perspective, development of the Security and Control Strategy would typically be done within the ITIL Service Design Lifecycle phase. To ensure the information assets of the business are secure, the Security and Control Strategy should address the following recommended best practices contributing to the lifecycle management of the My Oracle Support Services portal:

  • Perform My Oracle Support Security Access Control Configuration - Define security requirements to secure user access to My Oracle Support based on users’ roles and responsibilities. Assess security risks and establish controls to mitigate identified risks. Establish policies, processes and procedures for managing My Oracle Support security controls to include data security and functional access. Establish plans for monitoring, reporting, and auditing for compliance.
  • View My Oracle Support Security Access Control Best Practices - View best practices to understand the My Oracle Support access control capabilities and how to use them to maintain security and segregation of duties.
  • View My Oracle Support Roles and Privileges Best Practices - View best practices to understand the My Oracle Support user roles and the responsibilities of the roles.
  • Complete Initial My Oracle Support Account Registration - Complete user registration for getting access to the My Oracle Support portal.
  • Log In To My Oracle Support to Verify Registration - Log in to the My Oracle Support Portal to verify registration.

TA.100 Develop System Management Procedures

System Management is the process by which the system is controlled, operated, maintained and monitored with allowances for additions and / or changes as necessary to meet business requirements. System Management procedures should be documented and available to ensure that the system is operated, maintained and modified in a controlled manner. My Oracle Support should be a key component of the System Management Procedures. From an ITIL perspective, defining requirements for System Management Procedures would typically be done within the ITIL Service Design Lifecycle phase. To ensure effective use of My Oracle Support, the System Management Procedures should address the following recommended best practices contributing to the lifecycle management of an Oracle investment.

  • Define Customer User Administrator Processes - Manage the My Oracle Support access control processes. The Oracle Customer User Administrator is responsible for securing the access and use of the data and services available through My Oracle Support. Establish management procedures for administering My Oracle Support and user system access. Establish plans for monitoring, reporting, and auditing for compliance.
  • View Customer User Administrator Best Practices - View best practices to understand the responsibilities of the customer user administrator. Understand how to use My Oracle Support functionality to maintain My Oracle Support security access control.
  • Identify Customer User Administrators – Define a System Management Procedure to ensure a minimum of two customer user administrators are assigned to manage user access to My Oracle Support. An overview of the roles and responsibilities of the Customer User Administrator can be found here.
  • View Working Effectively with Support Best Practices - View Working Effectively With Global Customer Support [ID 166650.1] best practices to understand how to effectively collaborate with Oracle support.
  • View My Oracle Support Reporting Best Practices - View best practices to understand how to use My Oracle Support reporting capabilities.
  • Establish Training for My Oracle Support Services - Establish training to understand how to use the capabilities of My Oracle Support services to manage Oracle investments. Oracle has recorded training sessions that will provide beneficial tips and information to help with specific situations.

TRANSITION (TS) PROCESS GUIDANCE

TS.040 Design Production Support Infrastructure

The Production Support Infrastructure involves the management and coordination of the implementation of the processes, and systems, which from an ITIL perspective, are documented during the ITIL Service Design Lifecycle phase. My Oracle Support should be a key component of the Design Production Support Infrastructure. In designing your production support infrastructure, ensure that you provide for a test environment that is a logical match for the production environment to allow reproduction and resolution of incidents. The Design Production Support Infrastructure task description provides guidance for the development and improvement of capabilities for transitioning the production support infrastructure into operations. Design Production Support infrastructure would typically be done within the ITIL Service Transition Lifecycle phase. To ensure effective transition to production operations, Design Production Support Infrastructure should address the following recommended best practices contributing to the lifecycle management of your Oracle investment:

  • Perform My Oracle Support Best Practice Recommendations - Establish awareness of My Oracle Support best practice recommendations. Include My Oracle Support best practice recommendations in the design of your production support infrastructure. The total cost of ownership of the Oracle investment is improved with the adoption of My Oracle Support best practices.
  • Establish Policies for My Oracle Support Health Check Usage - Apply best practice guidance to establish policies for using My Oracle Support health checks. Use of My Oracle Support health checks improves availability and minimizes the impact of service disruptions.
  • Establish Policies for My Oracle Support Patch Plans - Apply best practice guidance to establish policies for using My Oracle Support patch plans. Use of My Oracle Support patch plans improves availability and minimizes the impact of service disruptions.
  • View My Oracle Support Projects Best Practices - View best practices to understand My Oracle Support projects functionality and how My Oracle Support projects can be used to manage system health.
  • View Oracle Web Conferencing Best Practices - View best practices to understand how to use Oracle Web Conferencing to prepare for online collaboration with Oracle support.
  • View Oracle Communities Best Practices - View best practices to understand how to collaborate with Oracle communities. Oracle communities provide a valuable avenue for customers to share knowledge.

TS.052 Implement Production Support Infrastructure

The Implementation of the Production Support Infrastructure involves the building, testing and deployment of the Design Production Support Infrastructure. My Oracle Support should be a key component of the Implement Production Support Infrastructure. The Implement Production Support Infrastructure task description provides guidance to put into effect efficient repeatable build and test mechanisms to ensure effective transition to production. From an ITIL perspective, Implement Production Support infrastructure would typically be done within the ITIL Service Transition Lifecycle phase. Implement Production Support Infrastructure should address the following recommended best practices contributing to the lifecycle management of the My Oracle Support Services portal:

  • Implement Support Test Environment – Implement the design of your production support infrastructure. Validate the implemented test environment is a logical match for the production environment.
  • Verify External Internet Connectivity - All support services tasks depend upon having external internet access. Test My Oracle Support Portal to determine connectivity to external sources.
  • Install Adobe PDF Requirements for My Oracle Support – Install Adobe PDF requirements on end user desktops.
  • Configure Adobe PDF Requirements for My Oracle Support - Configure Adobe PDF settings on end user desktops.
  • Verify My Oracle Support Browser Desktop Installation - Confirm supported My Oracle Support browsers are installed, configured and available for use on end user desktops.
  • Configure Oracle Web Conferencing - Configure Oracle Web Conferencing settings on end user desktops to prepare for online collaboration with Oracle Support
  • Verify Oracle Web Conferencing Compatibility - Run the Oracle Web Conferencing compatibility test to confirm desktop browsers are configured properly and ready for collaboration with Oracle Support.
  • Configure Customer User Administrator My Oracle Support Account - Register administrators with My Oracle Support
  • Administer Customer Support Identifiers - Administer Customer Support Identifiers to ensure users have access to the currently licensed products within Oracle support services. Access to My Oracle Support is controlled through user assignments to Customer Support Identifiers.
  • Configure Default Customer Support Identifier Privileges - Configure the Default Privileges for the Customer Support Identifiers administered by the customer user administrator.
  • Administer My Oracle Support User Security Access - Administer Role-Based Access to My Oracle Support functionality in accordance to corporate security policy.
  • View My Oracle Support Personalization Best Practices - View best practices to understand the personalization capabilities of My Oracle Support.
  • Configure My Oracle Support Service Request Profiles - Configure the My Oracle Support service request profiles. The service request profiles serve as templates to streamline the create service request process. To configure this setting, go to the settings tab located at the left hand column, and select Service Request profiles under Personal settings.
  • Configure Hot Topic Email Preferences - Configure personalization settings to receive email messages to create awareness for new or updated topics of interest.
  • Configure My Oracle Support Dashboard Regions - Use the customize button in the Dashboard to configure specific details through various customizable regions that include Service Requests. The Dashboard tab provides you with a concise, personalized, dynamic view of your systems and interactions with Oracle Support.
  • Configure My Oracle Support PowerView Preferences - Configure PowerView to take advantage of the My Oracle Support global search filtering capabilities. To configure this setting, go to the PowerView settings located at the top-left of the Dashboard.
  • Configure My Oracle Support Alert Preferences - Subscribe to My Oracle Support alerts relevant to business responsibilities to keep current on information that can help mitigate service disruption. To configure this setting, go to the Knowledge Articles region and select the modify settings located at the top-right corner of the region.
  • Configure My Oracle Support Newsletter Preferences - Subscribe to Oracle support newsletters that are relevant to the business to keep current on information. Oracle support teams provide news and technical updates through newsletters.
  • Install Oracle Configuration Manager Per Established Policies - Install Configuration Manager per established policies for use with My Oracle Support if not already done so.
  • Configure Oracle Configuration Manager - Configure Oracle Configuration Manager to comply with business requirements.
  • Schedule Oracle Configuration Manager Collections - Schedule the collection of configuration data to track Oracle environment configuration details.
  • Execute Trial Run of Oracle Configuration - Execute a trial run of Oracle Configuration Manager to verify everything works as expected.
  • Configure My Oracle Support Health Checks - Configure My Oracle Support health check settings to align with business requirements. To configure this setting, go to the Dashboard and modify settings in the Health Recommendations region.
  • Configure My Oracle Support Patch Recommendations - Configure My Oracle Support patch recommendations to align with business requirements. To configure this setting, go to the Dashboard and modify settings in the Patch Recommendations region.
  • Configure My Oracle Support Projects - Configure projects to get assistance in managing complex support scenarios. Projects provide Oracle support a contextual view of the business. To configure this setting, go to the Dashboard and modify settings in the Projects region.
  • Subscribe to My Oracle Support Communities - Subscribe to communities within the Oracle domain that are of interest to enable instant connection to a network of peers, industry experts, and Oracle product specialists.
