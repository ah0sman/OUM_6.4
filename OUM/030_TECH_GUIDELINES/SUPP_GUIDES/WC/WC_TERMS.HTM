DEFINITIONS AND TERMS

Accessibility
Software is accessible when vision- and hearing-impaired users and others with handicaps can access the application using a screen reader or other Section 508-compliant tools.

Access Levels
When you create a security policy, you must select a default access level for it. A default access level is assigned to any Content DB member whom you do not explicitly add to the access list associated with a security policy. You create the access list for a security policy by granting or denying access to specific members, groups, or organizations. As part of this process, you must also select an access level for each member or group that you add to the access list.

Administrative Options
Administrative options allow a Content DB administrator to control global settings that affect the functionality of the Content DB Application for all users.

Administrator
A Content DB administrator is a user who is a member of an administrative group. Administrators can add new users and assign roles, and set global options.

Application Program Interface (API)
A traditional definition of the Application Program Interface (API) is that of an interface between the operating system and application programs. For example, an API may make it possible for programs that run under it to open windows and display message boxes. However, in the case of Content DB, the API gives access to all the functions in Content DB some of which are not currently exposed in the user interface. End users can develop applications to take advantage of the Content DB functionality. For additional information, please refer to the API User Manual.

Annotations
Functionality provided by third party partner viewers such as is provided by Informative Graphics that allow users to add markups to images or electronic documents, without changing the original document, with security controls limiting edit and view access right at the user level Examples of different types of annotations include redlining, comments, highlighting, and redaction.

Archive / Storage
Software that handles the utilization of the appropriate disk subsystem such as NetApp or EMC Centera based on the value and anticipated near-term need of the content.

Auto-Classification
Refers to the use of specialized software to automatically organize content according to its subject or context, based on rules or logic that help determine the proper classification. Partner products such as Kofax can auto-classify content during the scanning process with a combination of database indexes, that can also perform validating lookups and reading the content itself (OCR). The results are used to auto-classify the image into Content DB. Orchestria is a partner product that auto-classifies content using a method known a ‘meta-tagging’ using logic that ‘reads’ incoming content such as emails and extracts relevant information to index, classify and organize the content.

Audit Trail
A record of the changes made to content or user actions related to content. Examples including tracking who accessed content, when the content was changed, and the user, application, or workflow process that made the change. Often used for analyzing the processes within a working environment, or to trace activity on a particular asset for quality assurance or compliance purposes. Content and Records DB have extensive audit capability.

Authorization
Process of granting a user access permissions to create, edit, or delete content.

Author
The creator of a document is referred to as the author. The author has full access rights on the document.

B

Behaviour
A behaviour is the result of configuring the folder to present a default state and functions to users.

Briefcasing / Offline
Creating an editable, offline document collection set for remote use that can be synchronized back to the main repository. Not a feature currently supported by Content DB.

Browser
A browser is an application program that provides a way to look at and interact with all the information on the Web. Technically, a Web browser is a client program that uses the Hypertext Transfer Protocol (HTTP) to make requests of Web servers.

Business Process Management/BPEL
Technology that orchestrates and automates the execution of business processes that involve human and system interactions during which documents, information or tasks are passed from one participant to another for action, according to a set of procedural rules. The BPEL Process Automation technology includes a graphic design tools for mapping out a process definition (including its users, steps, rules, and conditions), and a process engine that executes the process.

Business Analytics
Within the ECM context, tools that provide data of the utilization, workflow, and process automation metadata and statistics, to enable analysis of efficiencies of the ECM tool and the processes it is enabling.

C

Caching
The automated process of staging or assembly of pages or content objects in advance of their required use normally onto a local server in order to improve performance in retrieval and presentation to users. Content DB does not support this feature.

Capture / Scanning
Third party technology that is provided by Partner products such as Kofax, for scanning and digitizing documents, recognizing key data on those documents, and preparing the images and data for import into Content DB. Capture also includes technology for ingesting content files (documents such as faxes, web content, records and email items). Capture also refers to the extraction of forms data from a form and metadata from content, for example, capturing the metadata properties from the Microsoft Office properties page or from within HTML page metatags.

Categories
Categories contain metadata about documents, and help organize documents added to a Content DB library. Each category can contain any combination of documents and subcategories, providing users with the flexibility to group information in a variety of ways. Users can link or add the same document to multiple categories. In addition, they can regroup documents by unlinking or removing them from a category. To organize documents users must create document folders that contain files. Document folders can also be categorized and provide a method to organize documents that rely on a traditional folder hierarchy.

Check-In and Check-Out
Functionality that allows a user to access a document in a repository and lock other users out of that document while the user is making modifications or changes; checking the document back in to the repository removes the lock and also may invoke the creation of a new version.

Classification
The method and process of organizing content according to its subject or context.

COLD/ERM (Computer Output to Laser Disk / Enterprise Report Management)
A third party application such as Xenos that allows the transfer or migration of data such as computer generated reports from disparate legacy systems into a single Oracle Content DB and Records DB repository in order to repurpose the legacy data into a usable, easily accessed format that can be mined for information or presented on the Web for end-user / customer self-service.

Collaboration
Defined as a working practice whereby individuals work together to a common purpose to achieve business benefit. Collaboration tools such as Collaboration Suite, often through the implementation of team “workspaces,” facilitate discussion through forums and discussion threads, joint development and review of content, faster completion of tasks, and process visibility. Collaboration Suite also includes messaging, persistent chat, and presence notifications. Collaboration Suite is typically needed, by end-users who control content access, as opposed to a centrally-coordinated and constrained environment.

Component
Components are information objects, self-contained information that, when compiled with other components, make up a document. They provide their own context. Components can be created with many different tools, including Microsoft Word, Excel, or PowerPoint, and can be designed in any number of different formats. The defining characteristic of a component is not its format, but its scope, or level of granularity that allows it to be reused in creating many different documents. The method of storing components is often referred to as Compound Document Management and Content DB does not at this time support this approach.

Content Integration
The ability for disparate content (unstructured) systems to integrate with each other using a common architecture or approach such as SOA. Also allows for structured content (meta data) systems such as e-Business to communicate with content systems.

Content Management
see ENTERPRISE CONTENT MANAGEMENT.

Container
An Oracle DB virtual folder object that forms part of a structural element that can only have an administrative role assigned and can only contain a library or another container.

Customized Templates
Customized templates are normally designed to meet the needs of vertical market solutions to save time by pre-populating libraries with a meaningful taxonomy, common documents, and other frequently used items without sacrificing the flexibility to respond to changing work scenario’s.

D

Destination Document
The document in which a linked or embedded object is inserted using Microsoft object linking and embedding (OLE). The document that contains the information used to create the linked or embedded object is the source document. When you edit a destination document, the information in the source document is not updated.

Digital Asset Management (DAM)
A third party application that provides a repository for the storage and retrieval of digital assets such as images, graphics, audio, and video. Content DB can act as the storage location for all content types however the unique dynamic rendering, management and re-purposing of digital assets is only provided through a partner application such as Sealed Media (Stellent) and Media Bin (Interwoven).

Digital Rights Management (DRM/e-DRM)
A system for protecting the copyrights of data circulated via the Internet by enabling secure distribution and/or disabling illegal distribution of the data. Typically, a DRM system protects intellectual property either by encrypting the data so that it can only be accessed by authorized users or by marking the content with a digital watermark or similar method so that the content can not be freely distributed.

Digital Signature
The digital equivalent of affixing a person’s signature to a piece of paper, indicating authorship or approval.

Directory Services
A terminology normally used by Microsoft to describe a LDAP style directory that assists in finding the location of objects, such as documents, users, metadata synonyms, etc., in a computer system. For the Microsoft version of Content DB the integration is done using the OID (Oracle Internet Directory).

Disposition Schedule
The lifecycle of a record includes a point at which it is either archived permanently or disposed of. The disposition schedule defines the timeline and procedure to be used leading up to a documents destruction. Multiple steps can occur prior to a documents final destruction which are all supported by Records DB. These include ‘retain’, ‘move’ to another location or folder, ‘destroy’ the document and meta-data, and trigger a ‘workflow’ action or step.

Document
Documents are information objects designed to be stand-alone, self-contained, and complete. Documents are not dependent on other content for their meaning to be understood. They provide their own complete context. Documents can be created with many different tools, including Microsoft Word, Excel, or PowerPoint, and designed in any number of different formats. The defining characteristic of a document is not its format, but the way it organizes information. Examples of documents might include formal reports and manuals. A document is well suited to creation by a single or multiple authors, material that is well established, material that is likely to have a long shelf life and to be of interest to a wider audience. Documents are typically written with a wider audience and a longer timeframe in mind.

Document Folders
Document folders can contain any combination of documents and subfolders, providing the flexibility to group documents in a variety of ways. Each document folder and subfolder can be assigned a unique security policy, allowing controlled access to its contents.

Document History
A document history panel lists operations that have been performed on the document according to user, date, version, and other customizable criteria. see AUDIT TRAIL.

Document Profile / Document Index
A document profile or index is a record of document-specific metadata that is entered by a user who imports a document to a Content DB library. Metadata fields can be used as search criteria in document searches. Document profiles can be created or modified by a Content DB administrator.

Document Management/Electronic Document Management (DM/EDM)
The computerized capture, indexing, search, and lifecycle management of electronic documents (such as Microsoft Office documents) into the Content DB repository. Usually used to manage electronic documents that may need to undergo revisions or that may have multiple versions associated with them as they are used in a business process. Management of component (compound) documents is a mandatory requirement (not supported by Content DB at this time). The Content DB Application allows documents to be organized and manipulated in a wide variety of document formats. Common functions include:

  • Organize documents in customized document folders and categories
  • Assign additional custom metadata to documents
  • Share documents with users by assigning the appropriate security policy to the document
  • Track document history and versions
  • Delete obsolete documents
  • Create document reviews

Desktop Applications
The “thick-client”, content-centric applications to enable users to create, search, view, and update content and metadata. Content DB provides this capability through an approach known as the ‘Oracle Drive’ or ‘O Drive’, which integrates with, for example, Microsoft Word and Windows Explorer (via WebDAV).

Download Document
In the Content DB system, downloading means exporting a file to a directory outside the Content DB repository.

E

E-Forms
Online form generation tool such as Adobe Forms that automates and collects, stores, and distributes information, usually with the objective of avoiding redundant manual data input processes. Usually simpler than building a full-blown business application. E-Forms are typically used as part of a repetitive business process in conjunction with BPEL or another third party business process tool to manage the workflow.

Email Management
The ability to capture and store individual users’ e-mail messages (and attachments) as official records, and to retrieve, retain, or dispose of them based on business rules, filters, and corporate policy.

Embedded Object
All or a portion of a Word document, Excel spreadsheet, or Power Point slide can be pasted into a destination document as an embedded object using Microsoft object linking and embedding (OLE). Embedded objects become part of the destination document and, once inserted, have no connection to the source file. Double-click the embedded object to open it in its native application.

Enterprise Content Management (ECM)/Enterprise Content Services (ECS)
A set of technologies that help manage and control how enterprises create, capture, track, organize, store and publish an organization’s unstructured content. Such technologies bring increased control over content while making it easier for users to find and retrieve the right content in the right format at the right time. The technologies are usually presented as a series of solutions or applications (ECM) and as a range of services (ECS) exposed through an API (see API). Examples of functional categories that fall under the ECM umbrella include capture, document management, imaging, digital asset management, records management, web content management, and portal technology. Oracle Content DB along with third party partners provides a partial set of ECM applications and services.

Error Handling
Error handling generally involves catching and either recovering from or throwing an exception to be caught by the invoking object.

Event Handler
An event in Content DB is a programmable function whereby an automated event is triggered as the result of a specific action occurring. For example a workflow event is triggered when a new document is added to a folder.

Extranets
Extranets are a system of semi-private Web pages and shared files. An example of this would be a partner-download site where current customers can go to download the latest documentation and patches; however, someone simply surfing the Internet would not be able to access the site. Extranets are commonly hosted outside the firewall.

F

File Plan
A file plan is part of Records DB and contains the rules that determine how long a document is to be retained for (the retention schedule) and the process that should be followed until its destruction (the disposition schedule). Multiple file plans can exist in the Records DB configuration and once a document is declared as a record it will inherit a file plan.

Full-Text Search
Content DB searches support searching the full text of documents as well as the document profile fields for specified text strings.

Group
A group is a list of users who are defined as a collective for the purpose of assigning identical access privileges to all members. Groups can be composed of internal or external users. Only members of administrative groups can perform administration tasks such as adding users and modifying templates.

HTML
HTML, or Hypertext Mark-up Language, is the language used to create pages for the World Wide Web. You can use HTML to create hyperlinks as well as the text formatting. To view HTML, you will need a browser.

HTML Connectors
HTML Connectors support custom parameters that can be inserted into the HTML during page generation. The parameters can be defined for both administrative and per instance usage.

HTML Connectors Worklets
HTML Connector Worklets are Java classes that generate HTML for HTML Connectors.

Imaging
Ability to capture, index, manage, store, and display document images. Usually refers to the digitization of paper documents, and the indexing and import of those images into a managed repository for use in business processes or for archival.

Index
Profile data that describes a document. see METADATA, AUTO-CLASSIFICATION and DOCUMENT PROFILE.

Import Document
Uploading a file to a Content DB library. For example, a user can import documents to a folder on a Content DB page. When a file is imported, the user enters metadata on a document profile form.

Instant Messaging
Instant Messaging is part of Collaboration Suite and is an on-line conversation that takes place between just two people.

Intranets
Intranets are a company’s internal Web facilities. Intranets are hosted behind a firewall and are used only by a company’s employees.

J

JAMES (Java Apache Mail Engine Server)
JAMES is a 100% pure Java SMTP, POP3 Mail server, and NNTP News server designed to be a complete and portable enterprise-mail-engine solution based on currently available open protocols. JAMES is also a mail-application platform. The JAMES project hosts the Apache Mailet API and provides an implementation of this mail application platform API.

JAMES is based upon the Apache Avalon application framework. (For more information about Avalon, please go to http://jakarta.apache.org/avalon).

Java Authentication and Authorization Service (JAAS)
JAAS can be thought of as having two different parts: authentication and authorization. Authentication refers to an identity check (exactly like a login screen that asks for a name and password) while authorization refers to what an authenticated user can or cannot do (similar to the concept of access control).

K

Knowledge Management
Knowledge Management is the formal process of attempting to capture and manage a business’s intellectual property or assets. Generally, knowledge management involves collaboration across an organization to organize information assets.

L

LDAP (Lightweight Directory Access Protocol)
Lightweight Directory Access Protocol, or LDAP, is an open Internet standard. LDAP defines four basic models, which include an information model, a naming model, a functional model, and a security model. LDAP aims to make it possible for almost any application running on virtually any computer platform to obtain directory information, such as e-mail addresses and Public Keys. Because LDAP is an open protocol, applications need not worry about the type of server hosting the directory.

Library
A library is a Content DB folder that acts as a workspace that can contain folders and documents. Administrator defined access control roles such as creator and consumer can be assigned at this level.

Library Manager
An administration role that enables the user to assign rights to users and is able to define the configuration of the library and the workflow routing schema.

Link Document
A document that is stored in a Content DB library can be added to a folder on a Content DB page using the Link command on the folder’s Action menu. The link that appears in the folder is a reference to the document.

Link Types
Content DB allows a user to select a specific type for a link that is added to a link’s list, for example, a Content DB page, a URL to a Web site, or an email.

Linked Object
All or a portion of a Word document, Excel spreadsheet, or Power Point slide can be inserted into a source document as a linked object using Microsoft object linking and embedding (OLE). Linked data is stored in the source document. The destination file displays a representation of the linked data or an icon link, and stores the location of the source document.

M

Metadata
Metadata is data about data, the background information that helps users or search engines search for documents more effectively. Having more metadata enables users to search more effectively for and find specific documents. Metadata can include data about a particular document’s contents or structure. Metadata has the ability to make use of time stamps, tags, or user keywords defined by the document’s creator.

N

Native Connectors
Native Connectors expose the full internal functionality of the application and can do virtually anything the Content DB Application supports. They use the same native interfaces as the built-in items. Native Connectors can provide access to external data sources complete with custom property pages and views. They can also add functionality to existing Content DB objects. This type of connector requires extensive Java programming knowledge and requires XML knowledge for the development of page layouts.

Notification Engine
The concept of a notification engine supports various types of notification to enhance collaboration. The engine sends reminder-based and event-based notifications in the form of emails or messages and supports both system-defined and application-defined events. Additionally, the Notification Engine provides support for immediate or delayed delivery of server-side rules when an event is triggered. This feature is not currently provided in Content DB however could be considered as part of a consulting engagement.

O

Objects
Grady Booch describes objects as having state, behavior, and identity. What this means is that objects can have internal data and methods. Objects are also unique, meaning that they can be distinguished from other objects. Generally, an object is any item that can be individually selected and manipulated. This can include shapes and pictures that appear on a display screen as well as less tangible entities. In object-oriented programming, for example, an object is a self-contained entity that consists of both and manipulates data.

OLE
OLE (Object Linking and Embedding) is a framework for creating and manipulating compound documents. Content DB supports the embedding and linking of Word, Excel, and Power Point documents in a container document.

Operator / User
The operator or user of a document is the user to whom the document is currently checked out or with whom it is in use.

Operators / Query
To construct a query based on more than one term, operators must be inserted between the terms. Operators are codes that represent logic applied to a search. For example, Boolean operators such as AND, OR, NOT (or AND NOT), and NEAR are used to work with True/False values.

Owner
The creator of a Content DB content is referred to as the owner. The owner has full access rights to the page or page item.

P

Portable Document Format (PDF)
An Adobe Systems file format to which documents in nearly any format can be converted, retaining the fidelity of the original document. PDF makes it possible for formatted documents to appear on the recipient’s monitor or printer as intended, irrespective of the user’s operating system or output device used.

Portal
A browser-based gateway to a variety of content resources, repositories, and applications. Historically used in the context of Internet content sharing (i.e. web portals), but more increasingly used to describe a means of delivering enterprise wide content to internal users, i.e. become the de facto user interface for content consumers.

Process Automation / BPEL
Technology that manages and automates the execution of business processes that involve human and system interactions. The Oracle process automation tool is based on BPEL, which can be fully integrated with Content DB and includes design tools for mapping out a process definition (including its users, steps, rules, and conditions), and a process engine that executes the process.

Privilege
Privileges govern the actions that can be taken by a trustee. Privileges can govern library-level document folder-level and document-level actions and can be granted or revoked through the Privilege-setting properties. Privilege is not to be used as a synonym for permissions or rights.

Profile
Index data that describes a document (see Metadata). Also the act of indexing a document, most often via entry in an e-form.

Publish Document
When a document is published, it is full-text indexed and stored in a Content DB repository.

R

Realm
A realm is a storage location in the OID (Oracle Internet Directory) where users and groups are located. Multiple realms can be supported based on an organizations logical groupings. Content DB uses the OID realm to connect the Site to the users and groups in the realm. Realms are used as high level method of access control that separates users from discrete Content DB Sites.

Records Management
The management of records to meet business, compliance, audit, and accountability requirements. Records DB provides the ability to define a classification plan and retention schedule for business records (which can include documents, e-mail, and other file types), and to apply retention and disposition rules to those records in accordance with the organization’s records management policies and procedures. At this time Records DB does not include the management of paper records. Typically electronic records management systems are expected to comply with records management standards such as the Department of Defense (DoD) 5015.2. At this time Records DB is undergoing the process to comply with this standard.

Record Series
A record series is a ‘virtual folder’ with the purpose of containing multiple categories of record types. see RECORD CATEGORY.

Record Category
A record category contains meta-data attributes, security rights, the lifecycle rules and a destruction or disposition schedule for a collection of records. Multiple record categories would be managed as a record series.

Rendition / Rendering
The act of converting or “rendering” content in one format to another format (e.g. rendering a Microsoft Word document content as PDF), each separate, yet maintain a cross-reference link to maintain their existence as the “same” version.

Repository
A collection or library of documents or content, along with any metadata that describes the content and allows users to locate content. The Content DB repository uniquely uses the Oracle relational database to manage the metadata, unstructured content, and user access to the content.

Review and Approval
A specific set of workflow and collaboration tasks which makes the review of new and revised content easier and more efficient. Occasionally may utilize email, but more often an application environment specifically suited. Content DB currently includes Oracle Workflow (separate to BPEL) as a lightweight workflow engine built into Content DB for this purpose.

Roles
Roles are a pre-configured collection of permissions that are normally linked to users and groups. Content DB has a set of standard roles and hidden roles that are provided out of the box however an unlimited number of roles can be customized.

S

Search
Technology that allows users to submit queries to a repository, to receive a “hit list” of relevant content to which they have rights, and to retrieve the individual content objects. Content DB provides both fielded searching (to locate content based on metadata field values or ranges) as well as full-text searching (to locate content based on specific words that may exist within the content). By using Secure Enterprise Search (SES) as a separate component for advanced federated search – that is, searching across multiple repositories this is over-and-above the core functionality.

Search Folder
A Search Folder contains a list of links to documents that have been returned in a Content DB search.

Search Profile Index
A search profile is a list of document-specific fields that can be used as search criteria when performing a document search. In Content DB, search profiles contain all the search criteria and the results list filters documents based on the permissions and access controls assigned to the user.

Security
1. Technology that provides authorization and access to content for users and systems. Content DB does not install with pre-defined security policies however does include pre-defined roles that can be used as templates for defined access control for users. 2. In Content DB, security refers to methods and policies that enable authorized users to access content while unauthorized users are denied access. Security can exist on the network level (user authentication), the user or group level (roles), or the page, folder, or document level (access levels).

Security Policies
Security policies allow users to manage user access to content objects.

Server Cluster
The server cluster such as RAC can be described as a group of servers running Content DB services. The server cluster itself makes up the core entity of the server configuration. The middle tier can be separated and run on it’s own cluster.

Session Timeout Interval
The Session Timeout parameter determines how long a Content DB user can be inactive before being logged out by the system. This feature provides protection when users leave their workstation during a Content DB session.

Site
This is the top tier of a Content DB configuration. A site can contain one or many containers and/or libraries. Multiple sites can be supported however they require multiple realms in the OID (Oracle Internet Directory/LDAP Server) – see realms.

Source Document
This is the document that contains the information used to create a linked or embedded object using Microsoft object linking and embedding (OLE). When the source document is updated, the information in the linked object in the destination document can be automatically updated.

Start Page
The start page is the first Content DB page the user sees after login.

Structured Document
A collection of data items with implicit semantic relationships between the elements based on their organization. Simply, information that resides in fixed fields within a record or file, managed in the Oracle database.

Subfolder
A subfolder is a document folder that is contained within another folder. Multiple subfolders can be added to create a folder hierarchy for organizing documents on a Content DB page.

Syndication
The supply of content from a variety of sources, usually via a paid service subscription for reuse and integration with other content. A new functionality in some ECM tools, allowing users to subscribe to folders and change events, which is not currently provided in the core Content DB application.

T

Taxonomy
Taxonomy is the study of the general principles of scientific classification. This word is most commonly associated with biological or horticultural classifications according to natural relationships; however, it is also used in reference to technology to a hierarchical breakdown of information. Taxonomies help users organize content, navigate to specific content objects within the repository, and search for content. A good taxonomy enables the same content to be located and accessed via multiple paths.

Thesaurus
The ability to define synonym terms and other relationships, such as related or narrower terms for words, such that, either with the instantiated metadata for a given piece of content, or for a search term, the users likelihood of categorizing correctly and finding the desired content increases since the system provides synonym terms via a thesaurus capability.

Tomcat
Tomcat is the servlet container that is used in the official Reference Implementation for the Java Servlet and Java Server Pages technologies. The Java Servlet and Java Server Pages specifications are developed by Sun under the Java Community Process. Tomcat is developed in an open and participatory environment and released under the Apache Software License. For additional information about Tomcat, please refer to http://jakarta.apache.org/tomcat/index.html.

Trusted Logon / Single Signon
Defined as a logical relationship established between domains to allow pass-through authentication, in which a trusting domain honors the logon authentications of a trusted domain. User accounts and global groups defined in a trusted domain can be given rights and permissions in a trusting domain, even though the user accounts or groups don’t exist in the trusting domain’s directory. With regards to Content DB, a single signon is a trusted logon that is authenticated via the Content DB Server but through some third-party authentication system (SSO Server).

U

Unstructured Data
Data that does not reside in a fixed database or field. Examples of unstructured data include word processing files, spreadsheets, images, graphics, and web content.

User-Defined Security Policies
Content DB allows users to have security rights to content objects based on their role that contains a set of permissions such as create, delete or modify objects. This is part of a project definition and not provided out of the box.

User Profile
A user profile is a set of preferences that contains settings for default features that each user has as their working environment in Content DB. User default profile information can be configured only by a Content DB administrator however users can change their personal settings.

V

Version Control
Functionality that allows users to edit a document, retaining both the previous and the new versions, with an explicit version assignment. Content DB currently only maintains a major versioning scheme with no minor versioning scheme. In competing systems versioning can become quite complex with component documents where each component (e.g. chapter) has a unique version, and the collection of the components (e.g. full report) has its own assigned version.

View
Views determine how pages, folders, and searches display document information. Each document or search folder on a Content DB page has a customizable view that by default is presented in the documents host application. Third party viewers, supplied by Oracle partners such as Informative Graphics allows several hundred document formats to be viewed in a generic viewer that is not dependent on the the presence of a documents host application.

W

Web-based Distributed Authoring and Versioning (WebDAV)
A set of extensions to the HTTP protocol that allow users to create, edit, and move content on remote, often web servers.

Web Page
A web page, in contrast to a document, is not self-contained and relies on the context (navigation) of the web site it belongs to and the web as a whole. When a web page “wants” to refer to another piece of information, it links to it rather than copying and including it. This makes the reuse problem easy, but creates consistency and navigation problems.

Web Conferencing
Web Conferencing refers to the use of Web applications to allow people in different locations to view some type of presentation simultaneously. Web conferences focus on showcasing an event rather than the people participating in the event.

Web Content Management (WCM)
Provides a set of procedures for managing content from it’s creation throughout it’s lifecycle that is destined for publication to the Web. Content management across an enterprise also includes software-code management, integration with other enterprise systems, and the organization’s Web content.

Wizard
This is a series of dialogs that prompts the user to provide information necessary to complete a process. An example of this is the New Page Wizard.

Workflow / BPEL
Workflow is a term used to describe the tasks, procedural steps, organizations or people involved, required input and output information, and tools needed for each step in a business process. Oracle Workflow/BPEL includes design tools for mapping out a workflow or process definition (including its users, steps, rules, and conditions), and a workflow engine that executes the workflow. Oracle workflow manages both document-centric business processes with managing the document status during the process and includes advanced features and capabilities for controlling processes that require performing outside of Content DB.

eXtensible Markup Language (XML)
A standard language that enables the exchange of the content and format of documents among systems. It is extensible because designers can easily generate new definitions and schema understood by other systems. XML is increasingly changing the view of ECM systems managing “unstructured” content as XML, by definition, can provide structure to content files.

XSL
Extensible Style Language (XSL) is a style-sheet standard currently in development. XSL is similar to HTML in that it uses cascading style sheets to change document into formatted objects that are then presented via some type of media.
