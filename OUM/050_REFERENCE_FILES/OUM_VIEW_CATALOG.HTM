This catalog provides descriptions of the views currently available in OUM. Select a view that is applicable to your engagement. Each view provides access to specific method content. You can take advantage of several types of navigation: textual, method drop-down menu, and/or current page drop-down menu.

Cloud Views

  • CAS OUM (Cloud Application Services) - The CAS OUM (Cloud Application Services OUM) Solution Delivery Guide provides an overview of the approach, phases and tasks that define the Cloud Application Services OUM (CAS OUM) approach. The information in this guide enables project managers, and project team members, to better understand the full scope of Cloud Application Services implementations, and to plan and execute those engagements.

Manage Views

  • Manage - This view provides access to all the phases, activities and tasks associated with the Manage focus area (project management and program management), including supplemental content and reference files. This view is presented with an emphasis on the Manage activities with the tasks and work products organized in a work breakdown structure by phase and activity.

  • Manage Process - This view provides access to all the phases, activities and tasks associated with the Manage focus area (project management and program management), including supplemental content and reference files. The tasks and work products in this view are organized by process.

  • Scrum - This view provides a conceptual look at managing a Scrum-based project using the Oracle Unified Method (OUM) as the method and source of work products. It brings together in one place all the OUM tasks associated with managing a Scrum project as well as all OUM Scrum supplemental content.

Implement Views

  • CAS OUM (Cloud Application Services) - The CAS OUM (Cloud Application Services OUM) Solution Delivery Guide provides an overview of the approach, phases and tasks that define the Cloud Application Services OUM (CAS OUM) approach. The information in this guide enables project managers, and project team members, to better understand the full scope of Cloud Application Services implementations, and to plan and execute those engagements.

  • Requirements-Driven Application Implementation - This view provides support for implementing commercial-off-the-shelf (COTS) packaged application products, or product suites, such as Oracle E-Business Suite or PeopleSoft Enterprise, etc. This view reflects a requirements-driven approach to implementing packaged applications in that it is based on identifying requirements early in a project through interviews, process modeling workshops, etc., and crafting a business solution based on those requirements during the course of the project.
  • Solution-Driven Application Implementation - This view provides support for implementing commercial-off-the-shelf (COTS) packaged application products, or product suites, such as Oracle E-Business Suite or PeopleSoft Enterprise, etc. This view reflects a solution-driven approach to implementing packaged applications that leverages a pre-defined business solution (e.g., Oracle Business Accelerators) as the proposed business solution and tailoring that solution to the client's specific requirements during the project.
  • Software Upgrade - This view provides support for upgrading Oracle software products, including middleware, database, enterprise application products and Business Intelligence solutions. The view reflects an approach of upgrading from an existing software release to a new release, including the review and potential rationalization of any existing custom extensions.
  • Technology Full Lifecycle - This view provides access to the material required for full lifecycle technology projects. This view provides support for custom development, fusion middleware, and technology implementations. Additional support is provided through Technical Guidelines.
  • Business Intelligence (BI) - This view focuses on those OUM activities and tasks that are typically leveraged for custom Business Intelligence engagements, as well as engagements involving commercial-off-the-shelf (COTS) Business Intelligence offerings, such as Enterprise Performance Management (EPM) and other BI applications.

  • WebCenter - This view supports WebCenter project implementations. Oracle WebCenter is the user engagement platform for social businesses, connecting people and information. It brings together a portfolio of portal, web experience management, content, social and collaboration technologies into a single product suite.

  • Enterprise Security - This view provides access to the material required for using OUM on Enterprise Security projects. The supplemental guidance is provided to assist in tailoring project plans, based on the security domains defined by the International Information Systems Security Certification Consortium (ISC)2™. You may also use this view to understand the work done by security architects during Enterprise Security implementation projects.

  • Implement - This view provides access to all the phases, activities and tasks associated with the Implement focus area, including supplemental content and reference files.

  • Implement Core Workflow - This view provides the core tasks within the Implement focus area. This view should serve to accelerate the understanding of OUM by new practitioners and help to keep project teams focused on these tasks.

  • Implement - Example Work Products - This view provides a series of example work products, in OUM format, based on the SkiNow case study used in the OUM training courses.

Envision Views

  • Enterprise Architecture

    • TOGAF - This view provides support for The Open Group Architecture Framework (TOGAF). TOGAF is an industry standard architecture framework that can be used freely by any organization wishing to develop an Information Systems Architecture for use within that organization.
  • Strategy - This view provides access to the tasks used to form a strategy or approach to technical IT domains in the enterprise as a whole. This view is based only on the tasks within the Envision focus area. It allows the enterprise to explore technical approaches that could be utilized to resolve specific business problems. The Strategy view may be used to develop an approach around Business Intelligence, Service-Oriented Architecture, Security and many other domains. This Strategy view is generic. Used as a foundation, these tasks accomplish initial discovery and requirements gathering.

  • Enterprise Optimization Roadmap - This view is used to assist consultants who are leading or participating in this core Oracle Business Consulting service offering. This offering is designed to help organizations maximize the value of their Oracle investment and establish a clear focus on business results throughout the technology lifecycle. The view provides access to a predefined set of OUM material appropriate for this type of engagement.

  • Envision - This view provides access to all the phases, activities and tasks associated with the Envision focus area, including supplemental content and reference files. Envision concerns IT related issues at the enterprise level from an organizational, governance and architectural point of view. If not already started, Envision can be initiated at any time and should be continued for as long as the enterprise exists. Envision spans the whole project lifecycle of current and future IT projects. This view supports pre-sales, and Enterprise Architect disciplines.

  • Envision - Example Artifacts - This view provides a series of example Envision artifacts and models.

Business Process Management (BPM) Views

  • Business Process Management (BPM) in OUM - This page describes the context of BPM views in OUM and provides links to all the Business Process Management (BPM) views in OUM.

    • Business Process Management (BPM) Roadmap - This view is based on the BPM Roadmap described in the IT Strategies from Oracle (ITSO) guidelines and reference architectures library, Creating a BPM Roadmap. A BPM Roadmap provides the guidance to successfully adopt a business process oriented approach to building IT solutions to support the business. A well constructed BPM Roadmap defines projects that incrementally deliver BPM and provide business value at each step while minimizing the risks associated with BPM adoption. To build such a roadmap requires a solid understanding of the current situation with respect to BPM, a clear vision of what the BPM initiative needs to accomplish to be successful, and a structured approach to defining the iterations that comprise the path to successful BPM adoption.

    • Business Process Management (BPM) Project Engineering - This view is based on the IT Strategies from Oracle (ITSO) Business Process Engineering Practitioner Guide. The Oracle Business Process Engineering framework introduces a discipline to effectively deliver composite business application projects in a BPM environment. The framework encompasses the traditional analysis, design, implementation, and operational management and monitoring steps of software engineering, but addresses only the requirements specific to Business Process Engineering that are not covered by traditional methods and practices. In this way the Oracle Business Process Engineering framework can be used to simply augment existing practices.

Service-Oriented Architecture (SOA) Views

  • Service-Oriented Architecture (SOA) in OUM - This page describes the context of SOA views in OUM and provides links to all the Service-Oriented Architecture (SOA) views in OUM.

    • Service-Oriented Architecture (SOA) Roadmap - This view is based on the SOA Roadmap described in the IT Strategies from Oracle (ITSO) guidelines and reference architectures library, Creating an SOA Roadmap. A Service-Oriented Architecture (SOA) Roadmap provides the guidance to successfully adopt a service-oriented approach to building IT solutions to support the business. A well constructed SOA Roadmap defines projects that incrementally deliver SOA and provide business value at each step while minimizing the risks associated with SOA adoption. To build such a roadmap requires a solid understanding of the current situation with respect to SOA, a clear vision of what the SOA initiative needs to accomplish to be successful, and a structured approach to defining the iterations that comprise the path to successful SOA adoption.

    • Service-Oriented Architecture (SOA) Engineering Planning - This view is used to help organizations adjust or complement their project delivery methodology to support SOA execution. Many enterprises struggle with SOA adoption. One area of particular concern is the delivery of projects within their SOA. Many enterprises attempt to apply traditional software engineering methodologies (such as Unified Process, agile, waterfall, etc) that have not been adopted to handle the wider, cross-project reuse that must be addressed for projects participating in a SOA strategy. The SOA Engineering Planning view supports a service where the customer is taken through a series of facilitated workshops aimed at developing their own enterprise methodology for SOA projects.

    • Service-Oriented Architecture (SOA) Modeling Planning - This view is used for the planning of modeling activities in conjunction with SOA project delivery, and the development of a Motivational Model (as part of the Business Strategy Identification), process and data models, and service meta models. A service based on this view can be executed in a series of four optional areas. This view is designed to prepare organizations to incorporate modeling into their service engineering process. It should be used as a guideline for engagements that are delivered to organizations that want to plan for modeling in their SOA delivery process and define models in support of such efforts.
    • Service-Oriented Architecture (SOA) Governance Planning - This view is used to help organizations build a SOA Governance Model that manages the SOA key assets. Many enterprises struggle with SOA adoption, and one of the reasons is that the benefits of SOA do not come automatically. A company needs an appropriate SOA Governance Model that manages the SOA key assets such as Services, Reference Architecture and the Service Infrastructure Layer. Performing a planning engagement based on this view provides a roadmap for the adoption of SOA Governance tailored to the manner in which a given organization works. During the engagement, you work with the customer to capture relevant information about Governance structures and practices, facilitate the development of the Governance vision and work with the customer’s staff to develop the Governance Roadmap.

    • Service-Oriented Architecture (SOA) Reference Architecture Planning - This view is used to help organizations develop an architectural vision for SOA, and a roadmap to execute against that vision. The vision defines a set of principles and architecture to create consistency across the enterprise. The roadmap defines a set of incremental steps to progress a customer from their current state of SOA infrastructure to their architectural vision.

    • Service-Oriented Architecture (SOA) Application Integration Architecture Project Delivery - This view is used to execute SOA implementations that utilize AIA. Oracle Application Integration Architecture provides an open standards based framework for creating cross-application business processes that support the way you run your business today, while paving the way for your long term, strategic, business transformation plans. Its application independent framework enables you to utilize the applications of your choice to create composite business processes unique to your business, on a flexible service-oriented architecture.

    • Service-Oriented Architecture (SOA) Tactical Project Delivery - This view is used to execute projects following the tactical approach to SOA, that is, no enterprise-specific SOA strategy is used, but the project follows generic SOA principles.

    • Service-Oriented Architecture Project Delivery - This view is used to execute projects following the SOA approach, either as an initial project, or as subsequent projects. In this view, the focus is on delivery but it also provides the possibility to first plan the customer’s SOA strategy at the enterprise level.

    • Service-Oriented Architecture (SOA) Core Workflow - This view is used to provide a conceptual view of the SOA approach that is provided by OUM. This view is intended to describe how OUM supports SOA from Envision through Implement. This view is not used to deliver an engagement, but rather to describe at a high-level, the work that is done to prepare an enterprise for SOA. The view highlights the service engineering tasks of OUM and explains how they relate to each other.

Operate View

  • Operate - This view provides access to all the life cycle phases and processes associated with Information Technology (IT) service management. IT service management refers to the entire set of activities, directed by policies, organized and structured in processes and supporting procedures – that are performed by an organization or part of an organization to plan, deliver, operate and control IT services offered to customers. The Operate View is aligned with the Information Technology Infrastructure Library (ITIL®), and describes the services provided by Oracle Managed Cloud Services (OMCS), as well as other organizations involved in providing managed services to Oracle customers.
