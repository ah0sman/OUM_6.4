Resources

This page is intended to give you a single resource for some key links into valuable OUM content. No matter what page you are on in OUM, you can always come directly to this page by using the Resources button located in the header of most OUM pages.

  • About OUM - This document provides a complete description of what is included in OUM as well as software requirements, de-installation information, known problems, and contributors.

  • Navigate OUM - This document provides information about the structure of the method pack and how to navigate.

  • Method Overview - This document provides a detailed introduction to the method including the method scope, approach, structure, and key concepts, as well as links to the Envision, Manage,Implement and Operate focus area overviews.

  • Focus Area Overviews
    • Manage Overview
    • Envision Overview
    • Implement Overview
    • Operate Overview

  • View Catalog - This document describes each of the views supported in this release of OUM.

  • Process Overviews, Task Overviews and Templates
    • Manage Processes, Tasks and Templates
    • Envision Processes, Tasks and Templates
    • Implement Processes, Tasks and Templates

  • Phase End Milestones and Checklists - Use these links to access the Phase End Milestone Overviews and their associated checklists:
    • Lifecycle Objective Milestone Overview (LO)
    • Lifecycle Architecture Milestone Overview (LA)
    • Initial Operational Capability Milestone Overview (IOC)
    • System in Production Milestone Overview (SP)
    • Sign-Off Milestone Overview (SO)

  • Template User's Guide - Use this link to access the Template User's Guide that provides detailed instructions for using the current work product templates provided in OUM.

  • Oracle Method Template Engine (OMTE) Template User's Guide (PDF) - Use this link to access the RETIRED OMTE Template User's Guide that provides detailed instructions for using work product templates or work products created from templates prior to OUM 6.3. With Release 6.3, the Oracle Method Template Engine was retired. This User’s Guide is also retired. It is being made available as a reference document for use when working with work products that had been created with versions of OUM prior to Release 6.3. The Oracle Method Template Engine is no longer being supported or enhanced. Templates in OUM Release 6.3 no longer use the Oracle Method Template Engine. Refer to the Template User’s Guide above for instructions on working with templates from OUM Release 6.3.

  • OUM Project Workplan - Use this link to download a zipped file containing the OUM Project Workplan template and the Project Workplan User's Guide. The workplan template includes both Manage and Implement (delivery method) details.

  • OUM Primavera Project Workplan - Use this link to access a Primavera Project Workplan template. The workplan template includes both Manage and Implement (delivery method) details. You need Primavera Risk Analysis in order to open and update the workplan template.

  • Supplemental Guidance - This page provides links to supplemental guides, view-specific resources and samples, method cross references and other helpful resources.

  • Work Breakdown Structures (HTML) - Use the following links to access OUM work breakdown structures in html format:
    • OUM Manage
    • OUM Envision
    • OUM Implement
    • OUM Full Method

    Activities and Tasks WBS Views (HTML) - Use the following links to access OUM Activity and Task WBS views in html format:
    • OUM Manage
    • OUM Envision
    • OUM Implement
    • OUM Full Method

  • OUM Key Work Products - Use this link to access the Implement Key Work Products.

  • OUM Mappings - Use this link to access a single resource for mappings between legacy or acquisition methods and OUM as well as to mappings for OUM roles to tasks and other helpful OUM mappings.

  • Technique Index - Use this link to access an index of all OUM techniques.

  • Examples - Example artifacts (work products, deliverables, test scripts, etc.) are available from the "Templates and Tools" section of some Task Overview pages. Some examples can also be accessed from the Envision - Example Artifacts and Implement - Example Work Products view pages.

  • Envision Touch Points - Use this link to access a listing of touch points (potential prerequisites) from Envision work products to Implement tasks. Keep in mind that an Envision engagement does not always precede an Implement engagement and, therefore, these touch points are not always available as artifacts.
  • Project Roles - Use this link to view the OUM Project Roles.
  • Definitions / Terms - Use this link to view the OUM Definitions / Terms.
  • Bibliography - These publications and web sites are excellent sources of information you may want to explore more fully.
  • Comments and Questions - Use this link to contribute your thoughts, comments, ideas, and work products or artifacts back to the Global Methods team so that we may constantly improve this body of work.
