OUM

Mappings

This page gives you a single resource for mappings between legacy or acquisition methods and OUM, as well as other valuable mappings.

Manage Focus Area Mappings

  • Project Management Method (PJM) 2.6.1 to OUM Manage Mapping - This document provides a mapping of the legacy Project Management Method 2.6.1 to the OUM Manage tasks.
  • OUM Manage to Compass Mapping - This document provides a mapping from OUM Manage to the Compass acquisition management method.

  • Scrum to OUM Mapping - This document provides a mapping of Scrum activities to OUM phases and tasks, as well as some additional helpful information.

  • OUM Roles Mapping - This document provides a mapping of roles to Manage tasks.

Envision Focus Area Mappings

  • OUM Envision Mapping - This document provides a mapping of which OUM tasks are included in the various Envision views, as well as which tasks are applicable for Industry Reference Architecture or Enterprise Software Framework.

  • OUM Roles Mapping - This document provides a mapping of roles to Envision tasks.

Implement Focus Area Mappings

  • OUM to Application Implementation Method (AIM) Mapping - This document provides a mapping from OUM to the AIM legacy method.

  • CAS OUM (Cloud Application Services OUM) to the Application Implementation Method (AIM) Mapping - This document provides a mapping from the latest CAS OUM (Cloud Application Services OUM) approach to the AIM legacy method. It also provides a reverse mapping of AIM to CAS OUM.

  • OUM to AIM for Business Flows (ABF) Mapping - This document provides a mapping from OUM to the ABF legacy method.

  • OUM to Compass Mapping - This document provides a mapping from OUM to the Compass acquisition method.
  • OUM to EasiPath Migration Method (EMM) Mapping - This document provides a mapping from OUM to the EMM legacy method.

  • OUM to Hyperion BPM Workbench Mapping - This document provides a mapping from OUM to Hyperion BPM Workbench.

  • OUM to J.D. Edwards OneMethodology Mapping - This document provides a mapping from OUM to the J.D. Edwards OneMethodology.

  • OUM to Siebel Results Roadmap Mapping - This document provides a mapping from the OUM Requirements-Driven Application Implementation view to the Siebel Results Roadmap (RR) Method.

  • Compass Standard Upgrade to OUM Upgrade Mapping - This document provides a mapping from the Compass standard upgrade acquisition method to OUM.

  • Data Warehouse Management Fast Track (DWMFT) to OUM Mapping - This document provides a mapping from the DWMFT legacy method to OUM.

  • Hyperion BPM Workbench to OUM 5.1 Mapping - This document provides a mapping from Hyperion BPM Workbench to OUM 5.1.

  • OUM Roles Mapping - This document provides a mapping of roles to Implement tasks.
