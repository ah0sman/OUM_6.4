Envision Touch Points

The following provides touch points (potential inputs) of Envision work products to Implement tasks. Keep in mind that an Envision engagement does not always precede an Implement engagement and, therefore, these touch points are not always available as artifacts.

In some cases, the potential input will need to be satisfied by some other means, before the corresponding Implement task can be properly executed (for example, candidate services portfolio). In other cases, the touch point is an indication of the need to check to see if there is an enterprise level artifact that applies (for example, Standards and Guidelines).
