VIEW DESCRIPTION AND KEY COMPONENTS

Description

The Service-Oriented Architecture (SOA) Project Delivery view is used to execute projects following the Service-Oriented Architecture (SOA) approach, either as an initial project, or as subsequent projects. In this view, the focus is on delivery but it also provides the possibility to first plan the customer’s SOA strategy at the enterprise level. Three approaches are supported:

  • Strategic: an extensive SOA strategic planning exercise is carried out, equivalent to carrying out all the SOA planning views, followed by a project which executes on this vision.
  • Pragmatic: a set of enterprise-level tasks is performed to sketch out an initial SOA strategy, followed by a project based on this strategy. The set of enterprise-level tasks chosen will depend on the needs of the customer.
  • Tactical: no enterprise level tasks are carried out, and therefore no customer-specific strategy is used, but the project follows generic SOA principles.
