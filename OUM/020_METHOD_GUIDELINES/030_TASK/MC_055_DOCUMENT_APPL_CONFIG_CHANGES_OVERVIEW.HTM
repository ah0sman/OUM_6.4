MC.055 - DOCUMENT APPLICATION CONFIGURATION CHANGES

The purpose of this task is to update the current application setup documentation to reflect the changes associated with the upgrade to the new release. The new release may include new or changed functionality requiring additional or modified setup data. As you map business requirements to application features, you also begin to define the configuration you need to support the solution. Your solution may include a number of user-defined codes, system and application level parameters, and enabled features. During this task, you revise setup decisions made during the initial implementation, or previous upgrade cycle.

APPROACH

The Application Configuration Change Document records the parameters, user-defined codes, and other setups for each application that requires configuration changes, or additional configuration, as a result of new, or changed, functionality in the new release. Such configuration changes should be made to the upgraded environment following any post upgrade steps.

Pay close attention to the sequence of the setup steps listed in the document. In particular, the setup steps shown may assume that the applications are configured in isolation, and thus do not account for integrated or redundant setup steps for related or integrated applications.

Critical Setup Parameters

Application parameters do not all carry the same importance to the business. Some are more critical in determining how the system will be operated. For instance, within some applications the following parameters control significant processing features and functions:

  • Set of Books — an accounting ledger with a particular chart of accounts, functional currency and accounting calendar
  • Balancing Entity — a division or other business unit for which you prepare a balance sheet
  • Inventory Organization — a plant, warehouse, or other type of business unit designed to provide control and transaction functionality within one or more applications modules

Multi-Phase, Multi-Site Implementations

If the original implementation was multi-phase and multi-site, the Application Configuration Change Document is particularly susceptible to the risk of change from one site to another. For those setups that are critical to process execution (and not just defaults), it is imperative that they be managed closely to assure as much consistency as possible across sites and phases. Given the number of setups and the importance of integrated sequencing, close coordination and management is essential.

WORK PRODUCT DETAILS

Audience, Distribution and Usage

The Application Configuration Change Document is used:

  • to compose profile options, application options and user-defined codes that are to be added or revised as a result of new or changed functionality in the new release
  • to consolidate the setup parameters and codes of all applications impacted by the new release for centralized maintenance
  • to capture and communicate the new or revised application setup decisions for implementation in the Production Environment following the completion of post-upgrade steps

Distribute the Application Setup Documents as follows:

  • to all team members
