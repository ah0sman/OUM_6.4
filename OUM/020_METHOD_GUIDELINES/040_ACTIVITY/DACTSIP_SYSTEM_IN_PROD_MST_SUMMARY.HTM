D.ACT.SP - SYSTEM IN PRODUCTION MILESTONE SUMMARY

In OUM, each phase finishes with a well-defined milestone.  It is important to communicate the milestone to all the stakeholders since it is a point in time where critical decisions should be made and goals evaluated. Each phase has a main milestone. The System in Production milestone is produced at the end of the Transition phase.  At this point, the stakeholders decide if the goals and objectives set for the project have been met and if a new increment should begin.

Back to Top

OBJECTIVES

  • Prepare users for Production.
  • Implement Production support infrastructure.
  • Deploy system into Production.
  • Obtain acceptance.

  • D .ACT.CDT Convert Data - Transition
  • D.ACT.DCMRCCT Deploy Change Management Roadmap and Communication Campaign - Transition
  • D.ACT.CITA Conduct IT Alignment
  • D.ACT.TEUT Train End Users - Transition
  • D.ACT.FD Finalize Documentation
  • D.ACT.GP Go Production
