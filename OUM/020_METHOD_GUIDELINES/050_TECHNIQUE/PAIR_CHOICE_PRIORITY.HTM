PAIR-CHOICE PRIORITY

The purpose of this technique is to assist in prioritization of requirements. Use this technique whenever assistance is needed in establishing requirement and/or use case priority.

TECHNIQUE RESULTS

Pair-Choice Matrix - The Pair-Choice Matrix is used to help prioritize use cases and requirements. The purpose is to cause comparison of one item to another so that eventually each item has been compared once to every item being prioritized.

TECHNIQUE STEPS

No.
 Step Component Component Description
1
 Review the use cases that describe how the requirements will be satisfied. Prepare a Pair-Choice Matrix. To do this you record the number of each use case on the horizontal and vertical axis of the Pair-Choice Matrix. Use Case Number Row and Column Designators The use case number is recorded once above each column and again to the left of each row in the Pair-Choice Matrix
2
 Meet with key stakeholders for each functional area to prioritize. First Intersection Cell of the Pair-Choice Matrix Set up a working session to prioritize Use Cases using the Pair-Choice Priority technique.
3
 Compare use case 1 to use case 2 and record the higher priority use case number in the column row intersection cell of the Pair-Choice Matrix.   Record the higher priority use case in the intersection cell of the Pair-Choice Matrix.
4
 Continue to compare use case 1 to each subsequent use case asking "which one is the higher priority?" as you work your way across the rows and down each column of the Pair-Choice Matrix. Intersection Cell of the Pair-Choice Matrix
 Count Tally column to the left of the vertical use case number column on the pair choice matrix. If there is a tie between two or more use cases perform pair choice on just those use cases.
5
 Review the completed matrix and count the number of times each use case number appears in each of the intersection cells of the Pair-Choice Matrix. Use Case Priority Count Tally  
6 Highlight the use case that is the highest priority, second, third fourth, etc., and record on the MoSCoW List.   After the Pair-Choice Matrix is completed, identify the priority of the use cases and record in the MoSCoW List priority column.

APPROACH

This technique can be used when there is difficulty in establishing the priority of items, for example, use cases and business requirements.
